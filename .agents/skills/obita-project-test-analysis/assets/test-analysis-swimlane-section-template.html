{{章节号}}{{业务名称}}业务泳道图与测试关注点

本章节用于将 {{业务名称}} 的端到端流程拆解到不同责任方泳道中，并从节点、分支、状态流转和外部依赖四个角度沉淀测试关注点。

关注点 覆盖节点 测试目标 核心用例方向 预期断言
T1 入口参数 {{发起请求节点}} 验证入口参数、权限、前端约束与后端校验一致。 必填缺失、格式非法、越权访问、重复提交、边界金额或数量。 错误码、错误文案、HTTP 状态、页面提示与接口响应保持一致。
T2 主流程 {{业务处理节点}} 验证正常链路完成核心状态推进和关键数据写入。 构造最小成功数据、完整成功数据、幂等重放、关键字段映射。 数据库状态、响应字段、审计日志、消息或通知内容符合预期。
T3 外部依赖 {{下游交互节点}} 验证第三方、MQ、定时任务或异步回调的不确定性处理。 下游成功、下游失败、超时、未知态、重复回调、乱序回调。 本地状态不误推进，补偿或重试策略生效，异常日志可定位。
T4 异常分支 {{失败分支/拒绝节点}} 验证业务拒绝、系统异常和补偿失败的兜底语义。 校验拒绝、业务规则拒绝、数据库更新失败、并发状态冲突。 状态回滚或保持、失败原因落库、用户可见结果和运营可查结果一致。
