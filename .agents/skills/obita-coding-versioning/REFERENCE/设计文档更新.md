## 设计文档更新原则

* 系统设计文档作为系统设计快照，不保留设计过程中不同版本的设计方案。
* 系统设计文档不存放不同设计方案（A、B）的描述，只保留当前设计以及当前方案需要 **确定关注** 的技术实现逻辑，方便模型迭代方案时阅读。
* 如果系统设计文档中内容，和本次 SPEC 或 改动方案不一致的地方，明确更新为当前实现的设计方案。
