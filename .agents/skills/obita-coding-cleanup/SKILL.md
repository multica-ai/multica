# obita-coding-cleanup

这个目录记录值得沉淀、可能重复发生的错误案例。

## 代码清理
- 实现并验证通过后，必须执行 cleanup 流程。
- cleanup 需要读取本轮设计文档，按设计确认代码的兼容逻辑，如果用户已确认不需要保留旧逻辑，需要清理代码。
- 当代码改动涉及字段或接口行为变化时，cleanup 必须判断旧字段是立即清除还是短期兼容保留；如果选择兼容保留，必须记录 `retirementTrigger` 和 `retirementTiming`。
- 当数据库结构发生变化时，cleanup 必须确认变更前后的数据库结构、服务器迁移、本地初始化 SQL 与代码清理逻辑一致。
- 如果数据库表、字段、索引、初始化数据或查询路径被替换，cleanup 必须明确旧结构是否立即清除、短期兼容保留或延迟退役。
- 本次代码变更过程中，临时打印的日志代码确保删除。
