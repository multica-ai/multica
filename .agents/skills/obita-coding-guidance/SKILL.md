# obita-coding-guidance

## 指令说明
* 使用方法：当识别到用户意图为编码或者编写接口文档时，会自动触发此技能并加载 REFERENCE 文件夹中的一个或几个相关的参考文件作为规范约束，以帮助用户编写高质量的代码和接口文档。

### 参考规范说明
* 参考文件：在文件夹`{.Codex} 或 {.codex}/skills/obita-coding-guidance/REFERENCE`下，采用分层设计的规范文件

#### 1. 表现层：负责表现层编码规范
- `表现层-Controller实现spec.md`：表现层编码规范，此规范适用于HTTP协议交互的服务实现

#### 2. 应用层：负责应用层编码规范
- `应用层-接口定义spec.md`：应用层接口定义规范，此规范适用于`obita-llm-runtime/`下RPC（包含HSF、Dubbo等）协议交互的接口定义
- `应用层-接口实现spec.md`：应用层接口实现编码规范，此规范适用于RPC（包含HSF、Dubbo等）协议交互的接口定义的业务流程实现

#### 3. 业务层：负责业务逻辑编码规范
- `业务层-业务逻辑spec.md`：业务逻辑编码规范，封装领域内业务逻辑和业务规则。区别于 应用层接口实现编码规范，后者是应用层接口实现的业务流程编码规范

#### 4. 数据层：负责数据接入层编码规范
- `数据层-建表spec.md`：数据层建表DDL规范
- `数据层-持久化spec.md`：数据层mapper编码规范

#### 5. 适配层：负责依赖服务编码规范
- `适配层-服务依赖spec.md`：外部服务依赖编码规范

#### 6. 文档：负责文档生成规范
- `文档-接口文档生成spec.md`：接口服务协议文档生成规范，同一个需求的接口文档合并至同一个文档，并放在合适的目录下

#### 使用示例

* 示例1：识别到用户编写Controller服务，会自动加载 `表现层-Controller实现spec.md`
* 示例2：识别到用户编写应用层接口及实现，会自动加载 `应用层-接口定义spec.md`、`应用层-接口实现spec.md`
* 示例3：识别到用户编写业务逻辑，会自动加载 `业务层-业务逻辑spec.md`

### 任务流程
分析用户输入材料及上下文，当用户输入或上下文包含 `${变更名称}` 文件夹时（在 `.changes` 目录下），使用标准编码流程；否则，使用默认编码流程。

#### 标准编码流程
1. 任何编码任务开始前务必读取以下文件了解需求内容和任务实施清单
   - 读取 `../spec.md`：充分理解需求功能、技术设计、风险与依赖等目标内容
   - 读取 `../tasks.md`： 获取任务列表实施清单（如果没有则跳过）
   - 读取 `REFERENCE/技术栈和工程结构.md`：获取工程结构设计，使得代码设计符合工程架构设计
   - 读取相关编码规范文件：认真分析、研读编码规范，务必确保编码符合规范
2. 严格按照任务列表顺序实施任务，禁止遗漏或跳过任务项，如果没办法一次完成所有任务则提示用户继续实施
   - 务必及时更新任务状态：每个一级任务（不是更细粒度的子任务项）完成后**必须立即**更新任务状态，如：一级任务“1. xxx”全部完成后更新此任务下面的所有子任务状态为`- [x]`
   - 检查任务完成状态：当所有任务都完成时，**再次检查** `tasks.md` 任务状态是否都为`- [x]`
3. 相关文件存放路径，按`verioning skill`变更存放
   
#### 默认编码流程
- 读取 `REFERENCE/技术栈和工程结构.md`：获取工程结构设计，使得代码设计符合工程架构设计
- 读取相关编码规范文件：认真分析、研读编码规范，务必确保编码符合规范
