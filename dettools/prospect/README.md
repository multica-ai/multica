# Prospect deterministic tools

This folder stores approved-but-not-yet-promoted candidate dettools generated by
Stage 6 (`generate dettools for human-indicated situations`).

- Keep candidate files here until human approval + Stage 8 promotion.
- Candidate files should be named `*.go` and include `candidate`/`generated` suffixes,
e.g. `agent_install_retry_candidate.go`.
- Do not import prospect tools into active agents until promoted.

## Staging convention

- `*.go` under this directory are proposal artifacts, not production imports.
- Promotion action:
  1. Move chosen file to `dettools/`.
  2. Update `dettools/README.md` if this becomes part of the base catalog.
  3. Import with `multica dettool import-file`.

## Manifest

Each promoted/reviewed candidate is tracked in:
`dettools/prospect/manifest.json`

Keep `status` values explicit and machine-readable:
- `draft`
- `awaiting_human_approve`
- `candidate`
- `rejected`
- `promoted`
- `archived`

Stage 6 writes `candidate` entries with:

- `tool_name`
- `human_approve_ref`
- `owner`
- `source_cluster_id`
- `generated_at`
- `candidate_path`
- `test_path`
