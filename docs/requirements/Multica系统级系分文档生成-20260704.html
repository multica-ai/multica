Multica系统级系分文档生成

生成日期：2026-07-04

需求澄清

  • 基于当前仓库代码与现有架构文档，新生成一套系统设计分析文档，而不是修改旧文档。
  • 文档目标不是单一功能，而是覆盖整个 Multica 产品当前已实现的系统能力，并按模块分门别类整理。
  • 生成内容以代码现状为准，重点覆盖服务端、Web、Desktop、Mobile、共享核心包、数据库实体关系、核心接口、关键时序、状态流转、下游集成与测试设计。
  • 本次不以未来规划、待实现方案或纯视觉设计规范为主，而以当前系统如何工作为主。
  • 默认输出为 docs/system-design/需求系统分析/ 下的新 HTML 系分文档，并按章节结构组织内容。

变更范围

  • 新增需求澄清产物到 docs/requirements/，作为后续系统设计输入。
  • 新增系统设计 HTML 文档，内容来源于 server/、apps/、packages/ 以及现有产品/架构文档。
  • 需要梳理的重点模块包括：Workspace、Issue、Project、Agent、Runtime/Daemon、Skill、Autopilot、Chat、Inbox、成员权限、认证登录、CLI 与跨端适配。
  • 需要补充系统级视角下的接口定义、实现时序、ER 图、状态机、下游依赖、测试用例、影响面分析、讨论记录。

影响面分析

  • API：需要从服务端接口与前端调用关系中提炼系统级接口分组，避免把每个细枝末节 endpoint 都平铺进去。
  • 数据：需要从数据库实体与领域对象映射中抽出核心 ER，而不是穷举全部表。
  • 权限：需要体现 workspace 隔离、member/owner/admin 权限边界，以及 member/agent 多态主体模型。
  • 集成：需要体现 daemon、本地 CLI provider、MCP、文件存储、邮件/认证、WebSocket/实时同步等外部依赖。
  • 测试：需要以系统级测试视角组织，不等于逐个现有测试文件抄录。
