# Multica 飞书公网入口变更说明（中文）

本文档给飞书应用管理员与 Multica 维护者使用，说明本次公网入口调整后的现状、需要在飞书开放平台完成的配置变更，以及当前 HTTP 临时方案的限制。

## 1. 本次变更概览

Multica 已从 SSH 隧道访问切换为服务器内置 Nginx 的独立公网入口：

- 当前公网地址：`http://115.190.235.210:14000`
- 登录页：`http://115.190.235.210:14000/login`
- 回调页：`http://115.190.235.210:14000/auth/callback`
- 当前入口协议：`HTTP`
- 当前入口端口：`14000`

说明：

- `13789` 端口已经被现有 OpenClaw / Macaron Hub 占用，Multica 没有复用该端口
- Multica 当前通过 `14000` 单端口反代到内部服务：
  - 前端：`127.0.0.1:13030`
  - 后端：`127.0.0.1:13080`

## 2. 需要飞书管理员关注的配置项

### 2.1 回调地址

当前临时公网回调地址：

```text
http://115.190.235.210:14000/auth/callback
```

但是要注意：

- 飞书开放平台通常更适合使用 **稳定域名 + HTTPS** 的回调地址
- 当前这版是临时 HTTP 公网方案，主要用于先把 Multica Web 服务公网打通
- 如果飞书平台不接受 HTTP 公网回调，后续需要切到 HTTPS 后再正式修改白名单

### 2.2 仍需保留的权限项

飞书应用仍需确保以下权限已开通：

- `contact:user.base:readonly`
- `contact:user.email:readonly`

如果权限未生效，浏览器授权后仍会在 Multica 后端看到飞书资料不完整或被平台拒绝。

## 3. 本次代码行为变化

### 3.1 无邮箱飞书账号现在可以登录

此前如果飞书返回的用户资料中没有 `email`，Multica 会直接报错：

```text
Feishu account has no email
```

现在已经改成：

- 允许无邮箱飞书账号继续创建 Multica 用户
- `name` 使用飞书用户名
- `email` 自动生成占位邮箱

占位邮箱格式：

```text
feishu+<open_id>@users.multica.local
```

例如：

```text
feishu+ou_031becea1276c85e36a8d7f237a35c7b@users.multica.local
```

### 3.2 账号绑定原则保持不变

当前仍遵循这些规则：

- 同邮箱旧账号 **不自动合并**
- 首次飞书登录 **自动创建账号**
- 邮箱验证码登录 **继续保留为兜底**

## 4. 当前公网方案的限制

### 4.1 云侧网络尚需放行 14000

服务器上的 Nginx 已经监听 `14000`，但是否能真正从外网访问，还取决于：

- 火山云安全组是否放行 `14000/TCP`
- 是否存在额外网络 ACL / 防火墙策略

如果从外部访问 `http://115.190.235.210:14000/login` 超时，优先检查云侧入站规则。

### 4.2 飞书正式联调仍建议切 HTTPS

当前这次变更先完成了 Multica 的公网入口改造，但从飞书角度看，正式生产联调仍推荐：

- 使用稳定域名
- 使用可信 HTTPS 证书
- 将回调地址改为：

```text
https://<正式域名>/auth/callback
```

## 5. 后续正式切换 HTTPS 时要改什么

一旦切到正式 HTTPS 地址，需要同步更新：

### 5.1 飞书开放平台

- 登录重定向白名单
- 回调地址说明文档
- 如有审核材料，也要同步更新截图与回调地址

### 5.2 Multica 服务器 `.env`

至少更新以下字段：

```env
FRONTEND_ORIGIN=https://<正式域名>
MULTICA_APP_URL=https://<正式域名>
LOCAL_UPLOAD_BASE_URL=https://<正式域名>/uploads
ALLOWED_ORIGINS=https://<正式域名>
FEISHU_REDIRECT_URI=https://<正式域名>/auth/callback
```

### 5.3 前端构建产物

前端必须重新构建并重新发布 `.next`，否则浏览器端仍可能使用旧公网地址。

## 6. 推荐的管理员执行顺序

1. 在火山云安全组放行 `14000/TCP`
2. 验证 `http://115.190.235.210:14000/login` 可以从外部打开
3. 若飞书允许当前 HTTP 回调，再尝试把白名单改成：
   - `http://115.190.235.210:14000/auth/callback`
4. 如果飞书平台不接受，则转入下一阶段：
   - 上域名
   - 配 HTTPS
   - 再切飞书正式回调地址

## 7. 验证口径

### 7.1 Multica 侧

- 登录页可公网打开
- `/auth/feishu` 可被前端正常调用
- 无邮箱飞书账号可以自动创建占位邮箱用户

### 7.2 飞书侧

- 授权后能够回跳到 Multica 回调页
- 不再报权限缺失或回调地址不匹配
- 如果仍失败，优先记录：
  - 飞书错误码
  - 最终回调 URL
  - 浏览器控制台里 `/auth/feishu` 的响应错误
