# 单能力 Design

## 目标

- 为 7.2 给出明确的收敛方向：若未来重启，优先做“个人保存视图 / 对象书签”，而不是通用收藏中心。
- 明确 7.2 当前属于低优先级、非当前阶段目标。

## 非目标

- 不在当前阶段实现书签功能。
- 不做通用 URL 收藏或知识卡片库。
- 不允许执行 Agent 自行创造书签对象模型。

## 当前架构基线

- 当前入口：无。
- 当前核心逻辑：只有 `useIssueViewStore` 的本地视图持久化。
- 当前存储或状态：无 bookmark schema。
- 当前 UI 或接口：无。

### 代码证据

- `apps/workspace/src/features/issues/stores/view-store.ts` `useIssueViewStore`：说明最接近的现有能力是个人视图筛选状态持久化。
- `apps/workspace/src/router.tsx` `routeTree`：说明没有书签入口。
- `apps/workspace/src`、`server` `rg(bookmark|bookmarks)`：说明没有独立实现。

## 缺口定义

- 7.2 当前缺的是对象边界，而不仅是存储。
- 若不先收敛目标对象，书签很容易变成混合 issue、项目、URL、查询条件的杂项盒子。

## 方案与权衡

### 方案 A：通用收藏中心

- 做法：支持任意 URL、对象和文本片段收藏。
- 优点：看起来功能完整。
- 风险：直接滑向知识管理域，偏离当前产品主轴。

### 方案 B：个人保存视图 / 对象书签，未来若重启时采用

- 做法：优先围绕 issue 视图、项目或仓库对象做可命名书签，先从个人私有开始。
- 优点：能复用现有对象模型，也和 `view-store` 最接近。
- 风险：仍需新增 bookmark 实体与入口。

### 方案 C：当前阶段挂起，仅记录收敛方向，推荐

- 做法：把 7.2 标为 P3 / 非当前阶段目标，并明确未来只允许走方案 B。
- 优点：避免当前阶段做成边界模糊的收藏中心。
- 风险：短期内无功能交付，但避免错误投资。

## 推荐方案

选择方案 C，并把方案 B 作为未来唯一可接受的重启方向。

7.2 当前没有代码起点，也不在产品主轴内。若未来确实要做，必须把“书签”收敛为现有工作对象的个人保存视图或对象书签，不能直接扩成通用知识收藏域。

## 数据模型或状态模型

```text
future bookmark
├─ id
├─ object_type(issue_view | project | repository)
├─ object_ref
├─ title
├─ owner_user_id
└─ ordering
```

- 当前阶段不创建该模型。
- workspace 共享书签不在首阶段范围。

## 接口契约

### 输入

- 当前阶段无输入。
- 未来若重启，输入至少包括目标对象类型、引用、标题与排序。

### 输出

- 当前阶段输出仅为“挂起 + 收敛方向”。
- 未来输出应为个人书签列表与应用动作。

## UI 或交互流程

### 页面交互流

```text
当前阶段：无页面

未来若重启：
对象页面 / 视图
  -> Save as bookmark
  -> 输入名称
  -> 保存到个人书签列表
  -> 从 Utilities / Sidebar 快速打开
```

### 状态机

```text
[hypothesis]
   -> [scope-narrowed]
   -> [product-approved]
   -> [implementable]
```

### 数据变化流

```text
issue view / project view
   -> bookmark create action
   -> future bookmark storage
   -> bookmark list
   -> reopen original object/view
```

## 权限、边界条件、异常路径

- 谁可以使用：当前阶段无人可用。
- 哪些输入非法：任何“支持任意 URL / 任意文本 / 任意资源”的泛化设计都视为越界。
- 失败时如何处理：若未来缺少对象主键或视图快照，应阻止保存而不是生成不可用书签。

## 实现约束

- 不要实现通用收藏夹。
- 不要跳过对象类型收敛。
- 首阶段仅允许个人私有，不做 workspace 共享。

## 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 范围失控到知识管理 | 与当前主轴偏离 | 未来只允许对象书签/保存视图 |
| 缺少对象模型 | 书签不可打开 | 依附既有 issue / project / repo 对象 |
| 共享边界不清 | 权限复杂度上升 | 先做个人私有 |

## 验收检查

1. 7.2 在文档中被标为 P3 / 非当前阶段目标。
2. 文档明确未来若重启，只能走“个人保存视图 / 对象书签”方向。
3. 不出现通用收藏中心的伪设计。
