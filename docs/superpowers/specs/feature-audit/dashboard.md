# 功能审计总览台账

## 摘要

这个总览按照 `docs/功能列表清单.md` 的三级条目等权汇总。

| 模块 | 状态 | 完成度 | 说明 |
| --- | --- | --- | --- |
| 01 任务管理 | 部分完成 | 25 / 51，约 49% | 核心 issue 流程已存在，但归档和重复任务缺失 |
| 02 时间管理 | 部分完成 | 17 / 24，约 71% | 计时和番茄能力较强，但提醒体系偏弱 |
| 03 项目与标签管理 | 部分完成 | 11 / 15，约 73% | CRUD 已较完整，但隐藏、颜色、标签筛选仍缺 |
| 04 数据统计与分析 | 部分完成 | 3 / 20，约 15% | 目前更多是操作页，不是统计产品面 |
| 05 数据管理与同步 | 部分完成 | 2 / 16，约 13% | 只有窄范围导入和内部实时刷新 |
| 06 系统设置与个性化 | 部分完成 | 1 / 20，约 5% | 只有主题切换，其他大多缺失或不对齐 |
| 07 辅助功能 | 部分完成 | 1 / 13，约 8% | 只有番茄 note 残片，没有完整辅助功能域 |

## 优先级视图

### 最接近完成的模块

1. 项目与标签管理
2. 时间管理
3. 任务管理

### 缺口最大的模块

1. 系统设置与个性化
2. 辅助功能
3. 数据管理与同步
4. 数据统计与分析

## 接力入口

- 从 `docs/superpowers/specs/feature-audit/<module>/overview.md` 进入对应一级模块
- 用对应二级 `spec.md` 作为后续实现计划的权威范围
- 每次模块完成度变化后，都要回写本台账
