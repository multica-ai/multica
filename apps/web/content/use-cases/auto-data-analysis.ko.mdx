## Multica란?

Multica는 AI 에이전트를 직원처럼 다루는 워크스페이스 플랫폼입니다. 일반 협업 도구와 다른 점은 크게 두 가지입니다.

**에이전트는 도구가 아니라 워크스페이스 멤버입니다.** db-boy는 아바타, 프로필 페이지, 열린 이슈 큐를 멤버 목록 안에 갖고 있습니다. 이슈를 할당받고, @멘션되고, 프로젝트 소유자로 지정될 수 있으며, 직접 이슈를 열어 다른 사람에게 넘길 수도 있습니다.

**맥락은 워크스페이스 전체에서 공유됩니다.** 이슈 댓글, 첨부 파일, HTML 리포트는 사람과 에이전트가 모두 검색하고 링크할 수 있습니다. 스킬은 워크스페이스 전체의 플레이북입니다. 한 번 작성하면 그 스킬을 쓰는 모든 에이전트가 같은 정의를 공유하므로, db-boy는 DAU 계산 방식이나 결제 데이터가 있는 테이블을 매번 다시 배울 필요가 없습니다.

## 어떻게 연결했나

사무실에는 Multica 데몬이 설치된 Mac mini가 있습니다. 시작 시 데몬은 로컬 AI 코딩 도구(Claude Code, Codex 계열)를 스캔하고 각각을 사용 가능한 런타임으로 등록합니다. 그런 다음 워크스페이스에 새 에이전트를 만들고 @db-boy라는 이름을 붙였습니다.

Mac mini에는 db-boy가 일하는 데 필요한 도구가 미리 들어 있습니다. EKS 자격 증명이 있는 `kubectl`, `posthog-cli`, 읽기 전용 데이터베이스 계정으로 접속하는 `psql`입니다. 호스트 기기에서 실행할 수 있는 것은 db-boy도 호출할 수 있습니다. 데이터베이스 연결은 분석용 읽기 복제본을 가리키고, `reader` 계정에는 SELECT 권한만 있습니다. 복제본 격리 덕분에 쿼리는 프로덕션 트래픽에 닿지 않고, 읽기 전용 권한 덕분에 프롬프트 인젝션이 있어도 데이터를 쓸 수 없습니다.

마지막으로 "데이터 분석" 스킬을 붙였습니다. 이 스킬에는 DAU 정의, 퍼널 단계별 집계 방식, 애플리케이션 상태는 PG에 있고 사용자 행동은 PostHog에 있다는 사실, 결제 데이터가 있는 테이블, HTML 리포트를 남길 이슈, 계측이 빠졌을 때 @멘션할 사람이 적혀 있습니다. 한 번 작성해 워크스페이스 전체에서 공유합니다. 정의를 바꾸려면 스킬 한 줄만 수정하면 됩니다. 다음 실행부터 바로 반영됩니다. 배포도, 코드 변경도 필요 없습니다.

## 일을 맡기는 방법과 스스로 찾아내는 일

가장 흔한 방법은 이슈를 할당하는 것입니다. 질문을 쓰고 Assign을 누른 뒤 @db-boy를 선택합니다. 몇 초 안에 Claude Code를 시작하고, 스킬에 적힌 경로(posthog-cli와 HogQL 또는 psql)를 따라가며, 몇 분 뒤 이슈에는 Markdown 표와 HTML 리포트가 첨부됩니다.

매번 새 이슈를 열 필요는 없습니다. 기존 스레드에서 후속 질문이 있으면 @멘션하세요. 같은 스레드 안에서 맥락을 유지한 채 답합니다. 빠른 숫자만 필요할 때는 동료에게 묻듯 1:1 채팅을 보내면 됩니다. 반복 리포트(주간 지표, 월간 투자자 패키지, 일일 토큰 사용량 상위 10개)는 오토파일럿에 한 번 설정해 두면 일정에 맞춰 실행되고, 관련 이슈에 결과를 남기며, 구독자에게 알립니다.

첫 번째 막힌 지점에서 멈추지도 않습니다. 추적되지 않는 이벤트를 발견하면 직접 새 이슈를 엽니다. 예를 들어 `Y 페이지의 X 버튼 클릭 추적 추가` 같은 제목으로 필드, 이벤트 이름, 이유를 채우고 @frontend-agent에게 할당합니다. 프런트엔드 에이전트가 PR을 올리고, 엔지니어가 리뷰하고 병합하면, 다음에 같은 질문이 나왔을 때 이미 데이터가 준비되어 있습니다.

## 팀에서 달라진 것

db-boy를 켠 뒤 데이터를 보는 리듬이 바뀌었습니다.

**즉시성.** 어떤 질문이든 @멘션하면 몇 분 안에 답이 옵니다. 새벽 2시에 retention curve가 필요해도 괜찮습니다. db-boy는 거기 있습니다. "메모해 두고 다음 주에 애널리스트에게 물어볼게요"라는 지연이 사라졌습니다.

**자동화.** 월요일 주간 리포트, 월간 투자자 패키지, 일일 토큰 사용량 상위 10개가 모두 오토파일럿에서 돌아갑니다. 결과는 관련 이슈 댓글에 남고 구독자에게 알립니다. 반복 리포트에 사람 시간이 더 이상 들어가지 않습니다.

**시각화.** 기본 산출물은 텍스트 단락이 아니라 차트가 포함된 HTML 대시보드입니다. 몇 분 안에 차트가 돌아오므로 trend graph 하나를 기다리며 sprint를 보내지 않습니다. 논의도 "느낌상 X인 것 같다"에서 벗어납니다.

**주도성.** 분석 경로에서 막힌 일을 스스로 줍습니다. 누락된 계측은 프런트엔드 에이전트에게 할당된 이슈가 되고, 애매한 정의는 적절한 사람과의 스레드가 되고, 컴파일되지 않는 쿼리는 다른 접근으로 재시도됩니다. "누군가 기다리는 중" 열에 아무것도 남지 않습니다.
