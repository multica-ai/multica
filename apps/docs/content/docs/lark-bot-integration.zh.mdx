把一个 Multica 智能体连接成飞书 Bot 后，团队成员可以在飞书里私聊它、在群里 @ 它，也可以用 `/issue` 创建 issue。对话和执行记录仍会保存到 Multica 工作区。

每个 Bot 只对应一个智能体。需要在飞书中使用多个智能体时，分别为它们连接 Bot。

## 连接 Bot

智能体的所有者、工作区 owner 和 admin 可以发起连接：

1. 打开 **智能体**，选择要接入飞书的智能体。
2. 进入 **集成**，点击 **绑定到飞书**。
3. 用飞书扫描二维码并确认授权。
4. 回到 Multica。页面显示 **已连接到飞书** 后即可使用。

二维码为一次性凭证，过期后重新生成即可。已经连接的智能体不会再次显示扫码入口；需要改名或调整权限时，点击 **在飞书中管理**。

<Callout type="info">
目前只开放中国大陆版飞书的新连接入口。已有的国际版 Lark 连接仍可继续使用和管理。
</Callout>

## 第一次使用

成员第一次给 Bot 发消息时，会收到一个账号绑定链接：

1. 打开链接并登录 Multica。
2. 确认要关联的工作区账号。
3. 回到飞书，再发送一次消息。

绑定后，Multica 才能确认消息来自哪位工作区成员。非成员无法使用 Bot。

## 在飞书中交代工作

### 私聊

直接给 Bot 发消息即可。连续消息会保留在同一段 Multica chat 中，智能体的回复也会发回当前对话。

### 群聊

先把 Bot 加入群聊，再 `@Bot` 并写下要求。Bot 只会响应明确 @ 它的消息；执行时会结合当前会话中可读取的近期上下文。

### 创建 issue

发送：

```text
/issue 修复登录后的跳转问题
Safari 登录成功后仍然停留在登录页。
```

第一行成为标题，后面的内容成为描述。只发送 `/issue` 时，Multica 会尝试使用上一条消息作为标题。

## 管理连接

打开 **设置 → 集成** 可以查看工作区内已连接的 Bot。工作区 owner 和 admin 可以在这里断开连接；智能体所有者也可以回到该智能体的 **集成** 页面管理自己的 Bot。

断开后，Bot 不再接收新消息，已有的 chat 和审计记录不会随之删除。

## 自托管配置

Multica Cloud 无需额外配置。自托管部署需要先设置一个用于加密 Bot 凭据的密钥：

```dotenv
MULTICA_LARK_SECRET_KEY=<base64 编码的 32 字节密钥>
```

可以这样生成：

```bash
openssl rand -base64 32
```

重启 API 服务后，再从智能体的 **集成** 页面连接 Bot。请长期保存这个密钥；更换或丢失后，已有凭据将无法解密，需要重新连接。

`MULTICA_LARK_HTTP_BASE_URL` 和 `MULTICA_LARK_CALLBACK_BASE_URL` 只用于代理、测试等特殊部署，正常情况下不需要设置。

## 遇到问题

- **没有绑定按钮**：确认当前账号是智能体所有者、工作区 owner 或 admin，并检查自托管密钥是否已配置。
- **二维码失效**：关闭弹窗后重新发起连接。
- **Bot 要求绑定账号**：打开绑定链接并使用该工作区的 Multica 账号登录。
- **Bot 不执行**：检查智能体是否已归档，以及它使用的[运行时](/daemon-runtimes)是否在线。
