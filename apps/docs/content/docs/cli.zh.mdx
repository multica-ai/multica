Multica CLI 既负责连接本机运行时，也可以在终端中管理工作区、issue、智能体和自动化。这里介绍常用路径；当前版本支持的参数始终以命令自身的 `--help` 为准。

## 安装

**macOS / Linux**

```bash
curl -fsSL https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.sh | bash
```

安装了 Homebrew 时也可以运行：

```bash
brew install multica-ai/tap/multica
```

**Windows PowerShell**

```powershell
irm https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.ps1 | iex
```

确认安装成功：

```bash
multica version
```

## 第一次连接

连接 Multica Cloud：

```bash
multica setup
```

连接自托管实例：

```bash
multica setup self-host \
  --server-url https://api.example.com \
  --app-url https://app.example.com
```

`setup` 会保存服务地址、打开浏览器完成登录，并启动守护进程。完成后检查：

```bash
multica auth status
multica daemon status
```

如果只需要重新登录，不想覆盖其他配置，运行 `multica login`。没有浏览器的环境可以先在 Web 设置中创建个人访问令牌，再输入：

```bash
multica login --token
```

命令会在终端中提示粘贴令牌，避免把完整值留在 shell history 中。

## 选择工作区

列出并切换默认工作区：

```bash
multica workspace list
multica workspace switch <slug>
```

之后的命令会使用这个工作区。单次命令可以通过 `--workspace-id` 临时覆盖，也可以设置 `MULTICA_WORKSPACE_ID`。

邀请成员：

```bash
multica workspace member invite teammate@example.com
multica workspace member invite admin@example.com --role admin
```

## 常用 issue 操作

```bash
# 查看和搜索
multica issue list
multica issue get MUL-123
multica issue search "登录失败"

# 创建和修改
multica issue create --title "修复登录失败"
multica issue status MUL-123 in_progress
multica issue assign MUL-123 --to "Backend Agent"

# 评论和执行记录
multica issue comment list MUL-123
multica issue comment add MUL-123 --content "请先检查回归测试"
multica issue runs MUL-123
```

长描述或评论建议从 stdin 读取，避免处理换行和引号：

```bash
multica issue create --title "升级说明" --description-stdin < notes.md
multica issue comment add MUL-123 --content-stdin < review.md
```

查看某次执行的消息，或停止 task：

```bash
multica issue run-messages <task-id> --issue MUL-123
multica issue cancel-task <task-id> --issue MUL-123
```

## 智能体和 skill

```bash
multica agent list
multica agent get <agent-id>
multica agent create --help
multica agent update <agent-id> --help

multica skill list
multica skill get <skill-id>
multica skill import --url <skill-url>
multica agent skills add <agent-id> --skill-ids <skill-id>
```

导入时遇到同名 skill，默认会停止且不修改现有内容。按意图选择：

```bash
multica skill import --url <skill-url> --on-conflict overwrite
multica skill import --url <skill-url> --on-conflict rename
multica skill import --url <skill-url> --on-conflict skip
```

`overwrite` 只允许 skill 的创建者使用，并保留原来的 ID 与智能体绑定关系。

## 守护进程和运行时

```bash
multica daemon start
multica daemon stop
multica daemon restart
multica daemon status --output json
multica daemon logs --follow

multica runtime list
multica runtime rename <runtime-id> "Office Mac"
multica runtime usage <runtime-id>
multica runtime activity <runtime-id>
```

删除仍绑定活跃智能体的运行时时，命令默认会拒绝。`multica runtime delete <id> --cascade` 会同时归档这些智能体并取消活动 task，执行前应先确认影响范围。

运行机制和自定义 profile 见[守护进程与运行时](/daemon-runtimes)。

## 命令概览

| 命令 | 用途 |
| --- | --- |
| `issue` | 创建、更新、分配和搜索 issue；管理评论、订阅、标签、属性与执行记录 |
| `project` | 管理项目及其资源 |
| `label`、`property` | 管理工作区标签和自定义属性 |
| `agent`、`skill`、`squad` | 管理智能体、skill 和小队 |
| `autopilot` | 管理自动化、触发器和运行历史 |
| `workspace` | 创建、查看、切换工作区和邀请成员 |
| `repo` | 管理工作区仓库，并把仓库检出到本机 |
| `daemon`、`runtime` | 启停本机守护进程，查看和管理运行时 |
| `attachment` | 上传或下载附件 |
| `user profile` | 查看或更新当前用户资料 |
| `auth`、`login`、`setup` | 登录、查看认证状态和初始化连接 |
| `config` | 查看或修改当前 CLI profile 的本地配置 |
| `update`、`version` | 更新 CLI 或查看版本 |

`multica chat` 读取的是**智能体当前正在处理的外部聊天会话**，主要供聊天集成中的智能体使用；它不是用于浏览任意工作区对话的通用命令。

## ID 和输出格式

Issue 推荐使用 `MUL-123` 这样的 key，也可以传完整 UUID。Issue 不接受 UUID 短前缀。

其他资源的 `list` 命令通常显示可复制的短 ID，并支持 `--full-id` 输出完整 UUID。短 ID 不唯一时，CLI 会要求增加字符或改用完整 UUID。执行任务的短 ID 还需要通过 `--issue` 指明所属 issue。

许多命令支持结构化输出：

```bash
multica issue list --output json
multica agent list --output json
```

写脚本时使用 JSON，不要解析面向终端的表格。可以先运行 `multica <command> --help`，确认该命令支持的输出格式和分页参数。

## Profile 与配置

默认配置保存在 `~/.multica/config.json`。使用 `--profile <name>` 可以隔离另一套服务地址、令牌、默认工作区和守护进程状态：

```bash
multica setup self-host --profile staging \
  --server-url https://api.staging.example.com \
  --app-url https://app.staging.example.com

multica issue list --profile staging
```

命名 profile 保存在 `~/.multica/profiles/<name>/config.json`。查看当前值：

```bash
multica config show
multica config show --profile staging
```

<Callout type="warning">
CLI 配置文件包含可代表你访问 Multica 的令牌。不要提交到仓库、上传到日志或与他人共享。
</Callout>
