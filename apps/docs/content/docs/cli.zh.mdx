Multica CLI 把 Web UI 能做的事几乎全部搬到了命令行上（创建 [issue](/issues)、分配 [智能体](/agents)、启动 [守护进程](/daemon-runtimes) 等等）。这一页把所有顶级命令列出来，每条配一句用途。完整 flag 和示例用 `cs-workflow <命令> --help` 查。

## 认证入口

第一次用 CLI 时先登录，拿一个**个人访问令牌（Personal Access Token，PAT）**：

```bash
cs-workflow login
```

浏览器会自动打开，你在 Web 端同意后，CLI 把 PAT（`mul_` 前缀）保存到 `~/.multica/config.json`。此后所有命令都会自动用这个 PAT 认证。

<Callout type="tip">
CI / 无浏览器环境跳过浏览器流程：先在 Web 端 **Settings → Personal Access Tokens** 创建一个 PAT，然后 `cs-workflow login --token <mul_...>` 直接填入。
</Callout>

Token 类型的详细区分见 [认证与令牌](/auth-tokens)。

## 认证与初始化

| 命令 | 用途 |
|---|---|
| `cs-workflow login` | 登录并保存 PAT |
| `cs-workflow auth status` | 查看当前登录状态、用户、工作区 |
| `cs-workflow auth logout` | 清除本地 PAT |
| `cs-workflow setup cloud` | Multica Cloud 一键初始化（登录 + 装 daemon） |
| `cs-workflow setup self-host` | 自部署后端的一键初始化 |

## 工作区和成员

| 命令 | 用途 |
|---|---|
| `cs-workflow workspace list` | 列出你有权访问的所有工作区 |
| `cs-workflow workspace get <slug>` | 查看一个工作区的详情 |
| `cs-workflow workspace member list` | 列出当前工作区的成员 |
| `cs-workflow workspace update <id> --name "..." [--description "..."] [--context "..."] [--issue-prefix "..."]` | 修改 workspace 元数据（admin/owner 权限）。长文本可用 `--description-stdin` / `--context-stdin`。 |

## Issue 和 Project

<Callout type="info">
`list` 类命令（`cs-workflow issue list`、`autopilot list`、`project list` 等）表格里默认显示**可直接复制**的短 ID：issue 是 key（如 `MUL-123`），其余资源是 UUID 短前缀。下面表格里的 `<id>` 同时接受短 ID 和完整 UUID，所以典型用法是 `cs-workflow issue list` → 复制 key → `cs-workflow issue get MUL-123`。需要完整 UUID 时给 `list` 加 `--full-id`。
</Callout>

| 命令 | 用途 |
|---|---|
| `cs-workflow issue list` | 列出 issue（默认显示可复制的 issue key） |
| `cs-workflow issue get <id>` | 查看单条 issue（接受 issue key 或 UUID） |
| `cs-workflow issue create --title "..."` | 创建新 issue |
| `cs-workflow issue update <id> ...` | 修改 issue（状态、优先级、分配人等） |
| `cs-workflow issue assign <id> --agent <slug>` | 分配给智能体（立即触发任务） |
| `cs-workflow issue status <id> --set <status>` | 快捷改状态 |
| `cs-workflow issue search <query>` | 关键字搜索 |
| `cs-workflow issue runs <id>` | 查看 issue 上智能体跑过的任务 |
| `cs-workflow issue rerun <id>` | 给该 issue 当前的智能体分配人重新创建一条任务 |
| `cs-workflow issue comment <id> ...` | 嵌套：看 / 发评论 |
| `cs-workflow issue subscriber <id> ...` | 嵌套：订阅 / 取消订阅 |
| `cs-workflow project list/get/create/update/delete/status` | Project CRUD |

| 命令 | 用途 |
|---|---|
| `cs-workflow agent list` | 列出工作区的智能体 |
| `cs-workflow agent get <slug>` | 查看智能体配置 |
| `cs-workflow agent create ...` | 创建智能体 |
| `cs-workflow agent update <slug> ...` | 修改智能体 |
| `cs-workflow agent archive <slug>` | 归档 |
| `cs-workflow agent restore <slug>` | 恢复归档的智能体 |
| `cs-workflow agent tasks <slug>` | 查看智能体的任务历史 |
| `cs-workflow agent skills ...` | 嵌套：挂载 / 卸载 Skill |
| `cs-workflow skill list/get/create/update/delete` | Skill CRUD |
| `cs-workflow skill import ...` | 从 GitHub / ClawHub / 本机导入 Skill |
| `cs-workflow skill files ...` | 嵌套：管理 Skill 的文件 |

## 小队

| 命令 | 用途 |
|---|---|
| `cs-workflow squad list` | 列出工作区里的小队 |
| `cs-workflow squad get <id>` | 查看一个小队 |
| `cs-workflow squad create --name "..." --leader <agent>` | 创建小队（owner / admin）|
| `cs-workflow squad update <id> ...` | 修改名字、描述、instructions、队长、头像 |
| `cs-workflow squad delete <id>` | 归档（软删除）—— 同时把分配给小队的 issue 转给队长 |
| `cs-workflow squad member list/add/remove <squad-id>` | 管理小队成员 |
| `cs-workflow squad activity <issue-id> <action\|no_action\|failed> --reason "..."` | 队长智能体每轮结束时调用，记录 evaluation |

完整模型见 [小队](/squads)。

## 守护进程和运行时

| 命令 | 用途 |
|---|---|
| `cs-workflow daemon start` | 启动 daemon（默认后台；加 `--foreground` 前台跑）|
| `cs-workflow daemon stop` | 停止 daemon |
| `cs-workflow daemon restart` | 重启 daemon |
| `cs-workflow daemon status` | 查看 daemon 是否在线 + 并发情况 |
| `cs-workflow daemon logs` | 查看 daemon 日志 |
| `cs-workflow runtime list` | 列出当前工作区的 runtime |
| `cs-workflow runtime usage` | 查看资源使用情况 |
| `cs-workflow runtime activity` | 近期活动记录 |
| `cs-workflow runtime update <id> ...` | 更新 runtime 配置 |

## 杂项

| 命令 | 用途 |
|---|---|
| `cs-workflow repo checkout <url>` | 把 repo 拉到本地以供智能体使用 |
| `cs-workflow config` | 查看 / 修改 CLI 本地配置 |
| `cs-workflow version` | 显示 CLI 版本 |
| `cs-workflow update` | 升级 CLI 到最新版 |
| `cs-workflow attachment download <id>` | 下载 issue / 评论的附件 |

## 查完整 flag

每条命令都支持 `--help`：

```bash
cs-workflow issue create --help
cs-workflow agent update --help
```

v2 会给每条命令一个独立的详细 reference 页。
