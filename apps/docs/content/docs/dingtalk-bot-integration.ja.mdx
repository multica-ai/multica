任意の[エージェント](/agents)を DingTalk（钉钉）アプリボットに紐づければ、チームは DingTalk の中から直接それを使えます——ボットに DM したり、グループで @ メンションしたり、`/issue` と入力してアプリを開かずに [Multica イシュー](/issues)を起票したりできます。紐づけはスキャンして作成するフローです：DingTalk で QR コードを 1 つスキャンすると、アプリがあなた自身の組織内に社内アプリとして自動作成され、資格情報も自動で発行されます。

各ボットは 1 つの Multica エージェントと **1 対 1** で紐づきます。2 つ目のエージェントを紐づけると 2 つ目のアプリが作られます。1 つのエージェントが 2 つのボットを持つことはありません。

## この連携でできること

| 場所 | 動作 |
|---|---|
| **エージェント → 連携** | エージェント詳細ページには **連携（Integrations）** タブがあります。owner と admin はそこに **DingTalk に接続** が表示され、接続すると **DingTalk に接続済み** の状態に切り替わり、**接続を解除** ボタンと **DingTalk で管理** リンクが付きます。 |
| **ボットに DM** | ワークスペースメンバーが DingTalk の中でボットに直接メッセージを送ります。各会話はそのエージェントとの Multica [chat](/chat) セッションになり、エージェントの返信はメッセージとして返ってきます。 |
| **グループで @ メンション** | ボットを DingTalk グループに追加して @ メンションします。DingTalk がボットに配信するのはロボットを @ メンションしたメッセージだけで、グループの他のメッセージをボットが見ることはありません。 |
| **`/issue` コマンド** | `/issue <タイトル>`（本文を続けてもよい）を送ると、ワークスペースに新しい Multica イシューが作られ、あなたの名義になります。 |
| **`/new` コマンド** | それまでの会話コンテキストを破棄し、新しいエージェントセッションでメッセージを処理します。 |
| **`/unbind` コマンド** | あなた自身の DingTalk アカウントと Multica の紐づけを解除します。 |
| **「🤔思考中」リアクション** | メッセージが受け付けられると、ボットはそのメッセージに「🤔思考中」（考え中）のリアクションを付け、返信が届いたときに外します。 |

## エージェントを紐づける（owner / admin）

紐づけはスキャンして作成するフローです——アプリのシークレットをコピーする必要も、開発者コンソールで手動でアプリを作る必要もありません。

1. **Agents → あなたのエージェント** からそのエージェントを開きます。
2. **連携（Integrations）** タブを開き、**DingTalk に接続** をクリックします。
3. QR コードが表示されます。スマートフォンで **DingTalk → スキャン** を開き、組織内でのアプリ作成を承認します。（スキャンしづらい場合は、ダイアログ内の「DingTalk で開く」リンクも使えます。）
4. 承認が完了するとダイアログが閉じ、エージェントに **DingTalk に接続済み** と表示されます。アプリはあなたの組織の社内アプリとして作成され、資格情報は自動で発行されて保存時に暗号化されます。

<Callout type="info">
QR は使い切りで、短い時間が過ぎると失効します。承認する前に失効してしまったら、**もう一度スキャン** をクリックして新しいコードを取得してください。また、Lark 連携と異なり、スキャンしてもあなた自身の DingTalk アイデンティティは紐づきません——初めてボットにメッセージを送ったときに下記のルールで識別され、組織内のメンバーは通常自動でマッチします。
</Callout>

エージェントが接続されると、**DingTalk に接続** ボタンは接続済みの状態に置き換わります：片側に **接続を解除**、もう片側に **DingTalk で管理** リンクです。リンクは [DingTalk 開発者コンソールの社内アプリ一覧](https://open-dev.dingtalk.com/fe/app#/corp/app)を開きます。スキャンで作成されたアプリはそこにあり、権限・資格情報・リリースはすべてコンソールで管理します。（スキャン作成フローはアプリ個別のディープリンクを返さないため、入口はアプリ一覧ページです。）

## ボットを使う（メンバー）

### アイデンティティ：通常は全自動

ボットは「DingTalk の中のあなた」がどの Multica アカウントに対応するかを知る必要があります。識別は 2 段階です。

1. **組織ディレクトリでの自動マッチ。** あなたがその組織のメンバーなら、ボットは企業ディレクトリを通じてあなたの DingTalk アイデンティティを Multica アカウントに対応づけます——DingTalk で Multica にサインインしたアカウントは unionid で、メールで登録したアカウントはディレクトリプロフィールのメールアドレスでマッチします。マッチすれば自動で紐づき、何もクリックする必要はありません。
2. **フォールバック：バインドリンク。** 自動マッチに失敗すると、ボットは「アカウントを紐づけてください」というメッセージで返信します。リンクを開いて Multica にサインインすると、あなたの DingTalk アカウントが Multica メンバーシップに紐づきます。リンクは使い切りで **15 分間** 有効です。失効したら、もう一度ボットにメッセージを送れば新しいリンクが届きます。

<Callout type="info">
自動マッチは企業ディレクトリの参照に依存します。スキャンで作成されたアプリに連絡先（通讯录）の読み取り専用権限がないと自動マッチは失敗し、バインドリンクにフォールバックします——DingTalk 開発者コンソールでそのアプリに該当する読み取り専用権限を付与すれば復活します。
</Callout>

<Callout type="warning">
ボットを使えるのは **ワークスペースのメンバー** だけです。メンバーでない場合、ボットは返信しません——あなたのメッセージは破棄されます（監査のために破棄理由だけを記録し、内容は保存しません）。
</Callout>

### 対話とコマンド（`/issue`・`/new`・`/unbind`）

- **エージェントに何でも聞く** —— ボットに DM するか、グループで @ メンションします。会話は通常のエージェント chat セッションです。待っている間、あなたのメッセージには「🤔思考中」リアクションが付き、エージェントの返信はメッセージとして返ってきます。
- **イシューを起票する** —— `/issue Fix the login redirect` と送れば、Multica はワークスペースにそのイシューを作成し、「✅ 已创建 MUL-123 — Fix the login redirect」という確認を返します。タイトルの後ろに行を足せば、それが説明になります。`/issue` を単体で送ると、直前のあなたのメッセージがタイトルになります。コマンドは小文字の `/issue` でなければなりません。
- **仕切り直す** —— `/new`（続けて新しい質問を書いてもかまいません。例：`/new デプロイの話をしよう`）を送ると、エージェントはそれまでの会話コンテキストを破棄し、新しいセッションで応答します。
- **紐づけを解除する** —— 引数なしで `/unbind` を 1 行で送ると、ボットはあなたの DingTalk アカウントと Multica の紐づけを解除し、チャットで確認を返します。次のメッセージからは再び識別が行われます。

エージェントが **オフライン**（ランタイムが接続されていない）または **アーカイブ済み** の場合、ボットはメッセージを黙って破棄するのではなく、短いステータス通知で返信します。エージェントが忙しい（同時タスク数の上限に達している）場合は、メッセージがキューに入っており、枠が空き次第処理されることを知らせます。

## 管理と切断

ワークスペース全体の管理は **設定 → 連携 → DingTalk** にあります。

- **接続済みのボット** は、ワークスペース内のすべてのボットと、それぞれが紐づくエージェントを一覧表示します。この一覧はすべてのメンバーから見えます。
- **接続を解除** は **owner / admin 専用** です。解除するとボットは DingTalk メッセージの受信を停止します。インストール記録は監査のために保持され（「取り消し済み」バッジ付きで表示）、あとで同じエージェントから再スキャンして再インストールできます。

## 権限

- **紐づけ / 解除** にはワークスペースの **owner** または **admin** が必要です。member には接続済みボット一覧は見えますが、紐づけや解除の操作は見えません。
- **ボットとの対話** には、DingTalk アイデンティティが解決済み（自動マッチまたは手動バインド）のワークスペースメンバーであることが必要です。それ以外の人のメッセージは一律に破棄されます。
- この連携は破棄されたメッセージの本文を保存することはありません——監査のために破棄理由だけを記録します。

## セルフホストのセットアップ

Multica Cloud では連携はすでに利用可能です——このセクションは飛ばしてください。

セルフホストの場合、**保存時の暗号化キーを設定するまで DingTalk はオフ** です。このキーは、各アプリの client secret がデータベースに触れる前にそれを暗号化します。

1. 32 バイトのキーを生成し、API サーバーに設定します。

   ```bash
   openssl rand -base64 32
   ```

   ```dotenv
   MULTICA_DINGTALK_SECRET_KEY=<base64-encoded 32-byte key>
   ```

2. API を再起動します。キーを設定するまで、**設定 → 連携 → DingTalk** には「DingTalk 連携が有効になっていません」という通知が表示され、**DingTalk に接続** のエントリポイントは非表示のままになります。

<Callout type="info">
その他の環境変数はすべて任意のオーバーライドです。通常運用では未設定のままにしてください。

- `DINGTALK_OPENAPI_BASE` —— 新方式の DingTalk OpenAPI ホスト（デフォルト `https://api.dingtalk.com`）。
- `DINGTALK_OAPI_BASE` —— レガシーの DingTalk OAPI ホスト（デフォルト `https://oapi.dingtalk.com`）。
- `MULTICA_DINGTALK_REGISTRATION_BASE_URL` —— スキャン作成（device flow）の登録エンドポイントの上書き。ステージングやモック用。
- `MULTICA_DINGTALK_REGISTRATION_SOURCE` —— DingTalk が割り当てるパートナーのソースラベル。空なら送信されません。
</Callout>

## トラブルシューティング

### QR ダイアログのインストールエラー

インストールダイアログに表示されるエラーは、次の理由（`error_reason`）に対応します。

| メッセージ | コード | 意味と対処 |
|---|---|---|
| QR コードの有効期限が切れました。もう一度スキャンしてください。 | `expired` | QR には期限があり、承認前に失効しました。**もう一度スキャン** をクリックしてください。 |
| DingTalk 側で承認が失敗またはキャンセルされました。 | `install_failed` | DingTalk でキャンセルしたか、DingTalk 側でアプリ作成に失敗しました。再試行してください。ダイアログのグレーの小さな文字に DingTalk 側の具体的な理由が表示されます。 |
| DingTalk がインストールを拒否しました。もう一度お試しください。 | `dingtalk_protocol_error` | DingTalk API がプロトコルレベルのエラーを返しました。再試行し、続く場合はサーバーから DingTalk へのネットワーク到達性を確認してください。 |
| アプリは作成されましたが、資格情報の検証に失敗しました。もう一度お試しください。 | `credentials_check_failed` | アプリは存在しますが、新しい資格情報でアクセストークンを取得できませんでした。もう一度スキャンしてインストールしてください。 |
| インストール情報を保存できませんでした。このエージェントには別のアプリが接続されている可能性があります。 | `installation_conflict` | エージェントにはすでにインストールがあります。**設定 → 連携 → DingTalk** で古い方を解除してから、再度紐づけてください。 |
| インストールセッションが失効または失われました。もう一度スキャンしてやり直してください。 | `session_lost` | インストールセッションはサーバーメモリ上にのみ存在します（約 30 分）。再起動やタイムアウトで失われます。もう一度スキャンしてください。 |
| このワークスペースで DingTalk ボットをインストールする権限がありません。 | `forbidden` | インストールの途中であなたのロールが変わりました。owner / admin に続きを依頼してください。 |
| インストールに失敗しました。もう一度お試しください。 | `internal_error` | サーバー側の予期しないエラーです。再試行し、サーバーログを確認してください。 |

### 設定ページの状態

| 状態 | 意味 |
|---|---|
| **DingTalk 連携が有効になっていません** | サーバーに `MULTICA_DINGTALK_SECRET_KEY` が設定されていません。上記のセルフホストのセットアップに従ってください。 |
| **DingTalk ボットのインストールは近日公開** | 暗号化キーは設定済みですが、このサーバービルドではスキャン作成フローが有効になっていません（理由は起動ログにあります）。 |

### チャットの問題

- **グループでボットが反応しない** —— グループメッセージはロボットを @ メンションする必要があります。DingTalk はメンションのないメッセージをボットに配信しません。
- **DM でも返信がまったくない** —— そのワークスペースのメンバーでない可能性があります。メッセージは破棄されます（破棄理由だけが記録されます）。
- **バインドリンクが失効した** —— ボットに何かメッセージを送れば、新しいリンクが届きます。
- **組織メンバーなのに自動マッチしない** —— アプリに連絡先の読み取り専用権限を付与するか（DingTalk 開発者コンソール → 対象アプリ → 権限管理）、バインドリンクを使ってください。
- **「⚠️ 本次处理失败了」（処理に失敗しました）が届いた** —— その実行は失敗しました。時間をおいて再試行するか言い換えてください。エラーの詳細は Multica でそのエージェントのタスクを確認できます。
