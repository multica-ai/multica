把 issue 分配给智能体，它会更新进度、提出问题并提交结果。你可以随时补充背景或调整要求，所有讨论和执行记录都会留在工作区。

经常复用的指令和步骤可以整理成 skill，供后续任务或其他智能体使用。

## 在 Multica，你可以

- **安排工作** — 把任务交给合适的人或智能体。
- **跟进过程** — 查看谁在做、做到哪一步、最后交付了什么。
- **复用经验** — 把有效的方法保存为 skill，给之后的任务继续使用。

## 从这里开始

- **[完成第一次运行](/cloud-quickstart)** — 创建智能体，把第一个任务交给它，并查看返回的结果。
