Multica Desktop 提供 macOS、Windows 和 Linux 版本。它和 Web 版使用相同的账号与工作区数据，但会在本机自动管理守护进程，并为不同工作区保留各自的标签页。

## Desktop 和 Web 怎么选

| | Web | Desktop |
| --- | --- | --- |
| 打开方式 | 浏览器 | 安装桌面应用 |
| 工作区标签页 | 使用浏览器标签页 | 每个工作区保留独立的一组标签页 |
| 守护进程 | 需要单独安装并启动 CLI | 登录后由应用自动启动 |
| 更新 | 刷新页面 | 通过桌面应用更新 |

临时查看或在公共电脑上使用时，Web 更方便。经常使用 Multica、需要在多个 issue 或工作区之间切换，或者不想自己维护守护进程时，可以使用 Desktop。

两者可以同时登录。只要连接的是同一个 Multica 服务，看到的数据就是共享的。

## 安装

从 [Multica 下载页](https://multica.ai/download) 选择与系统和处理器架构对应的安装包：

| 平台 | 安装包 |
| --- | --- |
| macOS | `.dmg` |
| Windows | `.exe` |
| Linux | `.AppImage`、`.deb` 或 `.rpm` |

安装后使用和 Web 版相同的邮箱登录。进入工作区后，Desktop 会启动自己内置的 Multica CLI，并检测本机已有的 AI 编程工具。

<Callout type="info">
Desktop 内置的 CLI 只供应用管理运行时使用。如果你还要在终端中运行 `multica issue` 等命令，仍建议按照[快速上手的 Web 路径](/cloud-quickstart#2-通过-cli-连接电脑)单独安装 CLI。
</Callout>

## 桌面标签页

Desktop 会为每个工作区单独保存标签页。例如，你在工作区 A 打开了三个 issue，切到工作区 B 后会看到 B 自己的标签页；切回 A 时，之前的三个标签页仍然保留。

同一资源只会在当前工作区打开一次。标签页可以重新排序、固定、关闭，也会保留各自的前进、后退和滚动位置。

退出登录会清除本机保存的所有标签页，避免下一位登录者看到上一个账号留下的页面。

## 内置守护进程

登录后，Desktop 会为当前 Multica 服务创建一个专用 CLI profile，并用它启动守护进程。这个 profile 位于：

```text
~/.multica/profiles/desktop-<host>/
```

它不会读取或覆盖你在终端中使用的默认 profile。如果你同时手动启动了另一个守护进程，Multica 会把它们显示为不同的运行时。

可以在 Desktop 的设置中查看运行状态和日志。工具没有被检测到时，先确认它能在普通终端中运行并且已经登录，再重启 Desktop 的守护进程。

## 更新 Desktop

自动更新默认开启。应用会在后台检查并下载新版本，下载完成后可以立即重启安装，也可以等下次退出时安装。你也可以在 **设置 → 更新**中关闭自动检查或手动检查新版本。

不同操作系统和处理器架构使用各自的更新包。如果自动更新失败，可以从下载页获取对应架构的新安装包并覆盖安装。

## 连接自托管实例

Desktop 默认连接 Multica Cloud。连接自托管实例时，在用户目录创建 `~/.multica/desktop.json`：

```json
{
  "schemaVersion": 1,
  "apiUrl": "https://api.example.com",
  "appUrl": "https://app.example.com"
}
```

`apiUrl` 是 backend 的公开地址，必须使用 `http` 或 `https`。`appUrl` 是用于登录和生成可分享链接的 Web 地址。

WebSocket 地址通常不需要填写，Desktop 会根据 `apiUrl` 自动生成同一地址下的 `/ws`。只有 WebSocket 使用不同地址时才设置 `wsUrl`：

```json
{
  "schemaVersion": 1,
  "apiUrl": "https://api.example.com",
  "appUrl": "https://app.example.com",
  "wsUrl": "wss://ws.example.com/socket"
}
```

保存后重启 Desktop。配置文件存在但 JSON、版本或 URL 无效时，应用会显示配置错误，不会自动连接 Cloud。删除该文件并重启即可恢复默认的 Cloud 配置。

<Callout type="warning">
Desktop 只能连接浏览器和执行电脑都能访问的地址。远程自托管实例应使用 HTTPS，并正确代理 WebSocket；完整配置见[自托管快速上手](/self-host-quickstart#3-选择访问方式)。
</Callout>
