Desktop 和 Web 共用同一账号和工作区数据，任选一个开始即可：

| | Desktop | Web |
|---|---|---|
| 适合 | 日常使用，不想管理后台进程 | 直接在浏览器中使用 |
| 连接电脑 | 打开应用后自动连接 | 通过 CLI 连接 |

<Callout type="info">
开始前，请先在运行智能体的电脑上安装并登录至少一款[支持的 AI 编程工具](/providers)，例如 Claude Code、Codex 或 Cursor。Multica Desktop 自带连接程序，但不包含这些工具。
</Callout>

## 使用 Desktop

### 1. 安装并登录

从 [Multica 下载页](https://multica.ai/download) 获取 macOS、Windows 或 Linux 版本。安装后打开 Desktop，使用邮箱验证码或 Google 登录。

首次登录时，你可以完成或跳过几个简单的问题，然后创建一个工作区。Desktop 和 Web 的这部分引导相同。

### 2. 选择运行时

Desktop 会自动启动连接程序，并检测本机已有的 AI 编程工具。选择一个状态为"在线"的运行时，然后继续。

如果工具是刚安装的，点"刷新"重新检测。

运行时连接成功后，继续[完成第一次运行](#完成第一次运行)。

## 使用 Web

### 1. 登录并创建工作区

打开 [multica.ai](https://multica.ai)，使用邮箱验证码或 Google 登录。完成或跳过前面的简单问题，然后创建一个工作区。

### 2. 通过 CLI 连接电脑

进入"连接一台电脑"后，选择"通过终端连接"。在准备执行任务的电脑上安装 Multica CLI：

**macOS / Linux**

```bash
curl -fsSL https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.sh | bash
```

**Windows PowerShell**

```powershell
irm https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.ps1 | iex
```

然后运行：

```bash
multica setup
```

这条命令会打开浏览器完成登录，并启动连接程序。回到 Multica 的浏览器标签页，运行时出现后，选择一个状态为"在线"的运行时，再点"连接并继续"。

## 完成第一次运行

从这里开始，Desktop 和 Web 的步骤相同。

### 3. 把第一个 issue 交给 Multica Helper

进入工作区后，Multica 会创建一个名为 **Multica Helper** 的智能体，并显示几个入门选项。选择"简单介绍一下 Multica"，把它分配给 Helper。

Multica 会据此创建 issue、将 Helper 设为负责人并开始运行，无需额外配置智能体。

### 4. 查看进度和结果

分配后，页面会打开新建的 issue。执行日志显示运行状态，Helper 的回复显示在时间线中。

当执行日志中的状态变为"已完成"，并且时间线中出现 Helper 的回复，就表示运行成功。需要查看详细过程时，点"查看记录"。

## 遇到问题

| 现象 | 先检查什么 |
|---|---|
| 没有找到运行时 | 确认 AI 编程工具已经安装并能在终端中运行。Desktop 点"刷新"；Web 重新运行 `multica setup`。 |
| 运行时显示离线 | Desktop 保持打开；Web 运行 `multica daemon status`，未启动时运行 `multica daemon start`。 |
| issue 一直排队 | 确认智能体使用的运行时在线。运行时正在处理其他任务时，新任务会继续排队。 |

更多情况见[故障排查](/troubleshooting)。
