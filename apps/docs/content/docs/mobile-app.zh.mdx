Multica iOS 客户端目前还没有上架 App Store。你可以在 Mac 上从开源仓库构建 Release 版本，再安装到自己的 iPhone。Android 客户端暂未提供。

安装后的 App 连接 Multica Cloud，使用已有账号即可登录。工作区、issue、评论和执行记录都保存在服务端，不需要从 Web 或 Desktop 手动同步。

## 准备工作

你需要：

- 一台安装了 Xcode 的 Mac；
- 已在 **Xcode → Settings → Accounts** 中登录的 Apple ID；
- 一台通过 USB 连接的 iPhone；
- 在 iPhone 的**设置 → 隐私与安全性**中打开开发者模式；
- Node.js、pnpm 和 Git。

如果 Xcode 或真机开发环境尚未配置，请先完成 Expo 的 [iOS Device 环境配置](https://docs.expo.dev/get-started/set-up-your-environment/)。

## 安装到 iPhone

克隆仓库并安装依赖：

```bash
git clone https://github.com/multica-ai/multica.git
cd multica
pnpm install
```

然后从仓库根目录运行：

```bash
pnpm ios:mobile:device:prod:release
```

这会构建一个不依赖 Metro 的 Release 版本，并安装到已连接的 iPhone。第一次构建需要下载 iOS 依赖并编译 React Native，之后会复用 Xcode 缓存。

如果 iPhone 阻止打开应用，到**设置 → 通用 → VPN 与设备管理**中信任对应的开发者证书。

## 免费签名的 7 天限制

免费 Apple ID 签名的 App 只能使用 7 天。到期后，把 iPhone 重新连接到 Mac，并再次运行同一条构建命令即可续签。

Apple Developer Program 账号签名的有效期更长，也可以用于 TestFlight 和正式分发。无论使用哪种签名，服务端数据都不会因为重新安装或续签而丢失。

## 签名失败

如果 Xcode 提示 `No matching provisioning profiles found`，默认 bundle identifier 可能不属于你的 Apple ID。换成自己控制的反向域名后重新构建：

```bash
export EXPO_BUNDLE_IDENTIFIER_PROD=com.yourname.multica
pnpm ios:mobile:device:prod:release
```

这个值只需要在你的 Apple Developer team 中可用，不影响 Multica 账号或工作区。

## 更新

iOS 自助安装版不会自动更新。拉取最新代码、更新依赖并重新构建：

```bash
git pull --ff-only
pnpm install
pnpm ios:mobile:device:prod:release
```

## 连接自托管实例

Release 构建会把 API 地址写入 App。要连接自托管实例，先修改 `apps/mobile/.env.production`：

```dotenv
EXPO_PUBLIC_API_URL=https://api.example.com
EXPO_PUBLIC_WEB_URL=https://app.example.com
```

然后重新运行构建命令。iPhone 必须能访问这个 API 地址；使用局域网地址时，不要写 `localhost`，因为它在 iPhone 上指向手机本身。

<Callout type="warning">
修改 `.env.production` 会改变当前 checkout 的生产配置。只做个人构建时，不要把这个修改提交到仓库。
</Callout>

如果你要开发移动端、使用 Simulator 或连接 staging 环境，请查看仓库中的 [`apps/mobile/README.md`](https://github.com/multica-ai/multica/blob/main/apps/mobile/README.md)。
