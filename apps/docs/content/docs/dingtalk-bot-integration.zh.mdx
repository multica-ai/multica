把任意[智能体](/agents)绑定到钉钉应用机器人，团队就能在钉钉里直接使用它——私聊 Bot、在群里 @ 它，或者输入 `/issue` 直接创建一个 [Multica issue](/issues)，不用打开应用。绑定走扫码创建流程：用钉钉扫一个二维码，应用就作为企业内部应用自动创建在你自己的组织里，凭证自动颁发。

每个 Bot 与一个 Multica 智能体**一对一**绑定。再绑定一个智能体会创建另一个应用；一个智能体永远不会有两个 Bot。

## 这个集成能做什么

| 入口 | 行为 |
|---|---|
| **智能体 → 集成** | 智能体详情页有一个 **集成（Integrations）** tab。所有者和管理员能在这里看到 **绑定到钉钉**；绑定后会变成 **已连接到钉钉** 状态，带 **断开连接** 按钮和 **在钉钉中管理** 链接。 |
| **私聊 Bot** | 工作区成员在钉钉里直接给 Bot 发消息。每段对话都会成为该智能体的一个 Multica [chat](/chat) 会话，智能体的回复以消息形式发回来。 |
| **群里 @ 它** | 把 Bot 加进钉钉群再 @ 它。钉钉只会把 @ 机器人的消息投递给 Bot——它永远看不到群里的其他消息。 |
| **`/issue` 命令** | 发送 `/issue <标题>`（可附正文）会在工作区创建一个新的 Multica issue，记在你名下。 |
| **`/new` 命令** | 丢弃当前对话上下文，用一个全新的智能体会话处理这条消息。 |
| **`/unbind` 命令** | 解除你自己的钉钉账号与 Multica 的绑定。 |
| **「🤔思考中」表情** | 消息被接收后，Bot 会给你那条消息贴上「🤔思考中」表情回应；回复送达时自动撤掉。 |

## 绑定智能体（所有者 / 管理员）

绑定走的是扫码创建流程——不用复制任何应用密钥，也不用进开发者后台手工建应用。

1. 在 **Agents → 你的智能体** 打开该智能体。
2. 进入 **集成（Integrations）** tab，点击 **绑定到钉钉**。
3. 弹出一个二维码。用手机打开 **钉钉 → 扫一扫**，授权在你的组织内创建应用。（不方便扫码时，弹窗里也有「或点这里直接在钉钉中打开」链接。）
4. 授权完成后弹窗自动关闭，智能体显示 **已连接到钉钉**。应用已作为企业内部应用创建在你的组织里，凭证自动颁发并加密落库。

<Callout type="info">
二维码是一次性的，并且会在较短时间后过期。如果在授权前就过期了，点 **重新扫码** 获取新码即可。另外，与飞书接入不同，扫码本身不会绑定你的钉钉身份——第一次给 Bot 发消息时按下文的规则识别，组织内成员通常会自动匹配。
</Callout>

智能体连接后，**绑定到钉钉** 按钮会替换成已连接状态：一侧是 **断开连接**，另一侧是 **在钉钉中管理** 链接——它打开[钉钉开发者后台的企业内部应用列表](https://open-dev.dingtalk.com/fe/app#/corp/app)，扫码创建的应用就在那里，权限、凭证、发布都在后台管理。（扫码创建流程拿不到单个应用的详情页链接，所以入口是应用列表页。）

## 使用 Bot（成员）

### 身份识别：通常全自动

Bot 需要知道「钉钉里的你」对应哪个 Multica 账号。识别按两步走：

1. **组织内自动匹配。** 如果你是这个组织的成员，Bot 会通过企业通讯录把你的钉钉身份和 Multica 账号对上——用钉钉登录 Multica 的账号按 unionid 匹配，用邮箱注册的账号按通讯录资料里的邮箱匹配。匹配成功即自动绑定，什么都不用点。
2. **兜底：绑定链接。** 自动匹配不成时，Bot 会回一条「点击绑定」消息。打开链接、登录 Multica，你的钉钉账号就关联到你的 Multica 成员身份。链接一次性有效，**15 分钟**内使用；过期了再给 Bot 发条消息就会拿到新链接。

<Callout type="info">
自动匹配依赖企业通讯录查询。扫码创建的应用如果没有通讯录只读权限，自动匹配会失效并退回绑定链接——在钉钉开发者后台给该应用授予通讯录相关的只读权限后即可恢复。
</Callout>

<Callout type="warning">
只有**工作区成员**才能使用 Bot。如果你不是成员，Bot 不会回复——你的消息会被丢弃（仅出于审计目的记录丢弃原因，不保存消息内容）。
</Callout>

### 对话与命令（`/issue`、`/new`、`/unbind`）

- **随便问智能体** —— 私聊 Bot，或在群里 @ 它。对话就是一段普通的智能体 chat 会话；等待期间你的消息上挂着「🤔思考中」，智能体的回复以消息形式发回来。
- **创建 issue** —— 发送 `/issue 修复登录跳转`，Multica 会在工作区创建这个 issue，并回一条「✅ 已创建 MUL-123 — 修复登录跳转」确认。标题后面再加几行就是描述；单发一个 `/issue` 会用你上一条消息作为标题。命令必须是小写的 `/issue`。
- **重新开话题** —— 发送 `/new`（可直接带上新问题，如 `/new 换个话题聊聊部署`），智能体会丢弃这段对话此前的上下文，用全新会话处理。
- **解除绑定** —— 单发一行 `/unbind`（不带参数），Bot 会解除你钉钉账号与 Multica 的绑定并回复确认；之后再发消息会重新识别身份。

如果智能体**离线**（运行时未连接）或**已归档**，Bot 会回一条简短的状态提示，而不是悄悄丢掉你的消息；如果智能体正忙（并发任务已满），Bot 会提示你的消息已排队，空出来后立即处理。

## 管理与断开

工作区级别的管理在 **设置 → 集成 → 钉钉**：

- **已连接的 Bot** 列出工作区里每个 Bot 以及它绑定的智能体。这个列表所有成员都能看到。
- **断开连接** 仅限 **所有者 / 管理员**。断开后 Bot 停止接收钉钉消息；安装记录会保留以便审计（显示「已撤销」徽标），之后你可以从同一个智能体重新扫码安装。

## 权限

- **绑定 / 断开** 需要工作区**所有者**或**管理员**。成员能看到已连接 Bot 列表，但看不到绑定或断开的操作。
- **和 Bot 对话** 需要你是工作区成员，且钉钉身份已识别（自动匹配或手动绑定）。其余的人一律被丢弃。
- 对于被丢弃的消息，集成不会保存消息内容——只记录一个丢弃原因，用于审计。

## 自部署配置

在 Multica Cloud 上这个集成已经可用——可跳过本节。

自部署时，**在你设置好静态加密密钥之前，钉钉集成是关闭的**。这个密钥会在每个应用的 client secret 落库之前对其加密。

1. 生成一个 32 字节的密钥并设置到 API 服务器：

   ```bash
   openssl rand -base64 32
   ```

   ```dotenv
   MULTICA_DINGTALK_SECRET_KEY=<base64 编码的 32 字节密钥>
   ```

2. 重启 API。在密钥设置好之前，**设置 → 集成 → 钉钉** 会显示「未启用钉钉集成」提示，**绑定到钉钉** 入口也会保持隐藏。

<Callout type="info">
其余环境变量都是可选覆盖项，正常运行时保持不设置即可：

- `DINGTALK_OPENAPI_BASE` —— 钉钉新版 OpenAPI 主机（默认 `https://api.dingtalk.com`）。
- `DINGTALK_OAPI_BASE` —— 钉钉旧版 OAPI 主机（默认 `https://oapi.dingtalk.com`）。
- `MULTICA_DINGTALK_REGISTRATION_BASE_URL` —— 扫码创建（device flow）注册端点覆盖，用于 staging 或 mock。
- `MULTICA_DINGTALK_REGISTRATION_SOURCE` —— 钉钉分配的合作方来源标识；留空则不上报。
</Callout>

## 排查

### 扫码安装报错

安装弹窗里的报错对应下面这些原因（`error_reason`）：

| 提示 | 代码 | 说明与处理 |
|---|---|---|
| 二维码过期，请重新扫码。 | `expired` | 二维码有时效，授权前就过期了。点 **重新扫码**。 |
| 钉钉侧授权失败或已取消。 | `install_failed` | 你在钉钉里取消了授权，或钉钉创建应用失败。重试；弹窗里的灰色小字带有钉钉侧的具体原因。 |
| 钉钉拒绝了安装请求，请重试。 | `dingtalk_protocol_error` | 钉钉接口返回了协议级错误。重试；持续出现时检查服务器到钉钉的网络连通性。 |
| 应用已创建，但凭证校验未通过，请重试。 | `credentials_check_failed` | 应用建出来了，但用新凭证换取访问令牌失败。重新扫码安装。 |
| 未能保存安装信息：该智能体可能已绑定其他应用。 | `installation_conflict` | 该智能体已有安装记录。先在 **设置 → 集成 → 钉钉** 断开旧的，再重新绑定。 |
| 安装会话已失效或丢失，请重新扫码。 | `session_lost` | 安装会话只存于服务器内存（约 30 分钟），服务器重启或超时后即丢失。重新扫码。 |
| 你已没有在此工作区安装钉钉 Bot 的权限。 | `forbidden` | 安装中途你的角色变了。请所有者 / 管理员继续操作。 |
| 安装失败，请重试。 | `internal_error` | 服务器侧的意外错误。重试并查看服务端日志。 |

### 设置页状态

| 状态 | 含义 |
|---|---|
| **未启用钉钉集成** | 服务器没有设置 `MULTICA_DINGTALK_SECRET_KEY`。按上文自部署配置操作。 |
| **钉钉 Bot 安装即将上线** | 加密密钥已设置，但这个服务端版本没有启用扫码创建流程（启动日志里会有原因）。 |

### 对话问题

- **群里 Bot 不理人** —— 群消息必须 @ 机器人；钉钉不会把未 @ 的消息投递给 Bot。
- **私聊也没有任何回复** —— 你可能不是该工作区成员，消息被直接丢弃（仅记录丢弃原因）。
- **绑定链接过期了** —— 再给 Bot 发任意一条消息，会拿到一条新的绑定链接。
- **组织内成员没有自动匹配** —— 给应用授予通讯录只读权限（钉钉开发者后台 → 该应用 → 权限管理），或直接走绑定链接。
- **收到「⚠️ 本次处理失败了」** —— 这次运行失败了。可以稍后重试或换个说法；错误详情到 Multica 里打开该智能体的任务查看。
