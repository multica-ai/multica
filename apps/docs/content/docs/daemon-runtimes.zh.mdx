Multica 负责记录和协调工作，连接的电脑负责执行。电脑上的**守护进程**（daemon）领取 task，再调用本机安装的 AI 编程工具。

## 守护进程和运行时分别是什么

- **守护进程**是运行在一台电脑上的 Multica 后台程序。它负责连接服务器、发现本机工具、领取 task 和回传结果。
- **运行时**代表一个工作区可以使用的具体执行环境。它对应一台电脑，以及这台电脑上的一款 AI 编程工具或自定义运行时配置。

例如，一台电脑同时安装了 Claude Code 和 Codex，并连接两个工作区，守护进程会分别为两个工作区注册 Claude Code 和 Codex 运行时。重启守护进程会更新原有记录，不会为同一个组合不断创建新运行时。

## 工作发生在哪里

本地运行时调用的 AI 编程工具、这些工具自己的登录凭据，以及本地代码目录都留在连接的电脑上。Multica 服务器不会代替本地工具执行命令，也不会自动上传整个工作目录。

为了让团队能够协作，服务器会保存 issue、评论、智能体配置、task 上下文、执行记录和智能体写回的结果。这些内容可能包含智能体主动读取并写入回复的代码片段或其他项目上下文。

<Callout type="warning">
智能体的自定义环境变量保存在服务器侧，并在执行时发送给运行时。不要把"本地执行"理解为所有 secret 都只存在本机；自定义环境变量和 MCP 配置仍应按敏感信息管理。
</Callout>

## 启动守护进程

使用 Multica Desktop 时，应用会自动启动守护进程，不需要额外运行命令。

使用 Web、远程电脑或无界面环境时，先[安装 Multica CLI](/cli)，再运行：

```bash
multica daemon start
```

守护进程默认在后台运行。常用命令如下：

| 命令 | 作用 |
| --- | --- |
| `multica daemon status` | 查看守护进程和连接状态 |
| `multica daemon logs -f` | 持续查看日志 |
| `multica daemon restart` | 重启守护进程并重新检测本机工具 |
| `multica daemon stop` | 停止守护进程 |
| `multica daemon start --foreground` | 在当前终端中运行，便于调试 |

启动时，守护进程会检测 `PATH` 中受支持的 AI 编程工具，并为你有权连接的工作区注册运行时。如果一款工具刚刚安装或登录，重启守护进程即可重新检测。

守护进程至少需要检测到一款内置支持的 AI 编程工具才能启动。安装方法和可执行文件名见[安装 AI 编程工具](/install-agent-runtime)。

## 执行任务如何到达这台电脑

运行时注册后会持续保持连接。新 task 入队时，服务器会通知对应守护进程；守护进程也会定期轮询，作为连接中断后的补充。因此运行时在线且有空闲容量时，task 通常会立即开始。

守护进程每 15 秒发送一次心跳。服务端综合心跳与连接状态判断运行时是否在线；守护进程意外退出后，最迟通常会在约 3 分钟内显示为离线。

运行时离线时：

- 已经排队的 task 会等待运行时恢复，最多保留 2 小时。
- 正在执行的 task 会失败；符合条件的 issue 或对话 task 可以自动重试。
- 守护进程重新启动后，会重新注册运行时，并回收上一次没有正常结束的 task。
- 离线超过 7 天且没有活动智能体绑定的运行时，会被自动清理。

详细状态和重试规则见[执行任务](/tasks)。

## 并发限制

一台守护进程默认最多同时执行 20 条 task；每个智能体默认最多同时执行 6 条。实际并发取两者中较小的一项。

达到上限后，新 task 会继续排队。你可以在智能体设置中调整单个智能体的并发数，也可以通过 `MULTICA_DAEMON_MAX_CONCURRENT_TASKS` 调整整台电脑的上限。提高并发前，应先确认机器性能、工具账号额度和工作目录是否能够承受并行执行。

## 谁可以使用运行时

本地运行时默认是**私有**的：运行时所有者和工作区的 owner、admin 可以在它上面创建智能体。

运行时所有者或管理员可以把它改为**公开**。公开后，工作区中的其他成员也可以选择这条运行时；这不会分享底层 AI 工具的登录凭据，只是允许成员把智能体的 task 路由到这台电脑。

## 自定义运行时配置

如果团队使用内部 wrapper、固定版本的可执行文件，或需要为一款兼容工具追加固定参数，可以创建**自定义运行时配置**。

自定义配置不会增加新的通信协议。创建时仍需选择一种 Multica 已支持的协议族；命令本身必须与该协议兼容。

### 创建配置

只有工作区 owner 和 admin 可以创建、修改或删除自定义运行时配置：

1. 打开 **运行时**，进入已经安装该命令的电脑。
2. 点击**添加自定义运行时**。
3. 选择命令实际兼容的协议族。
4. 填写名称、命令和固定参数，然后保存。

配置会在工作区内共享。每台已连接电脑会自行查找这个命令；只有能在 `PATH` 中解析到它的电脑才会注册对应运行时。创建配置不会自动安装命令，也不会替其他成员完成工具登录。

这里填写的是可执行文件和参数，不是 shell 脚本。可以使用普通参数、引号和反斜杠转义，但不能使用管道、重定向、`&&`、`;`、反引号或环境变量展开。需要这些行为时，请先写一个 wrapper script，再把它作为命令填写。

如果 Desktop 启动的守护进程找不到终端里能够执行的命令，可以为当前电脑设置绝对路径：

```bash
multica runtime profile set-path <profile-id> --path /absolute/path/to/command
```

移除路径覆盖：

```bash
multica runtime profile unset-path <profile-id>
```

修改配置只影响之后领取的 task。删除配置前，需要先处理仍绑定在对应运行时上的活动智能体；只删除某台电脑上的运行时实例并不会删除配置，运行中的守护进程还会重新注册它。

## 运行时没有上线时

按以下顺序检查：

1. 运行 `multica daemon status`，确认守护进程已经启动。
2. 运行 `multica daemon logs -f`，查看登录、网络或工具检测错误。
3. 在同一运行环境中执行 `command -v <工具命令>`，确认守护进程能够找到它。
4. 打开 Multica 的**运行时**页面，检查目标电脑和对应的 AI 编程工具是否显示在线。
5. 工具刚安装、路径刚修改或配置刚更新时，运行 `multica daemon restart`。

仍未解决时，参见[故障排查](/troubleshooting)。
