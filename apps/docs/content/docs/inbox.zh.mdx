收件箱汇总与你有关的工作动态。它不是完整的活动日志，而是提醒你"哪里发生了需要关注的变化"；完整上下文仍然保存在对应的 issue 中。

## 哪些内容会进入收件箱

常见通知包括：

- issue 分配给你，或从你名下移走；
- 你订阅的 issue 出现新评论，或负责人、状态、优先级和日期发生变化；
- 有人在 issue 描述或评论中 @你；
- 你创建的 issue 或评论收到表情回应；
- 你订阅的 issue 中，智能体执行失败；
- 通过智能体创建 issue 的操作完成或失败；
- 你订阅的自动化创建了新 issue，或自动化因连续失败被暂停。

你对 issue 做出的普通修改不会再通知给自己。多条来自同一 issue 的通知会合并成一项，打开后可以查看这条 issue 最近发生的变化。

## 哪些人会自动订阅 issue

以下参与者会自动成为订阅者：

- issue 的创建者；
- 新的负责人；
- 发表评论的人；
- 在 issue 描述中被新增 @提及的人；
- 自动化中预先配置的订阅成员。

自动订阅不会因为负责人后来改变而自动取消。你可以在 issue 页面随时订阅或取消订阅；取消后，普通更新不再进入收件箱，但别人直接 @你时仍然可以单独通知你。

<Callout type="info">
子 issue 的状态发生变化时，也会通知父 issue 的订阅者。子 issue 的评论、优先级和日期变化不会向父 issue 冒泡。
</Callout>

## 智能体不使用收件箱

收件箱只面向工作区成员。即使智能体是负责人或订阅者，它也不会读取收件箱通知。

在评论中 @智能体、把 issue 分配给智能体或触发自动化，都会直接创建一次执行，而不是给智能体留一条通知。`@all` 也只通知成员。

## 处理通知

选择一条通知会把它标为已读。处理完后可以归档；归档不会修改或删除对应的 issue。

收件箱菜单还可以：

- 全部标为已读；
- 归档全部或全部已读；
- 归档状态为 `done` 或 `cancelled` 的 issue 通知；
- 打开"已归档"并恢复某条通知。

## 调整通知类型

打开**设置 → 通知**，可以分别开关：

- 分配；
- 状态变更；
- 评论与提及；
- 优先级与日期；
- 智能体活动。

关闭某一类后，相关变化仍然保存在 issue 中，只是不再创建收件箱通知。系统通知和浏览器通知控制的是桌面横幅，不会改变收件箱本身。

<Callout type="warning">
`@all` 会通知工作区中的所有成员。请只在确实需要所有人看到时使用。
</Callout>
