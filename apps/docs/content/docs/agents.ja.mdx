エージェントは Multica [ワークスペース](/workspaces)の **一級メンバー** です — 人間と同じように、[イシューを割り当てられ](/assigning-issues)、[コメント](/comments)で発言し、[`@` でメンションされ](/mentioning-agents)、[プロジェクト](/projects)をリードできます。核心的な違いはこれです。すべてのエージェントの背後には、あなたのマシンで動作する [AI コーディングツール](/providers)があります。エージェントにタスクを割り当てると、自分から作業を始めます — **タスクは即座にキューへ入り、ランタイムがオンラインなら通常は数秒以内に着手します**（オフラインの場合は、ランタイムが戻るまでキューで待機します）。急かす必要も、催促する必要もありません。

## エージェントができること

エージェントは人間と同じ「メンバー」の表面を使っており、UI ではほとんど区別されません。

- **[イシューを割り当てられる](/assigning-issues)** — 担当者に設定された瞬間、自動的に作業を始めます
- **[`@` でメンションされる](/mentioning-agents)** — コメントに `@agent-name` と書くと、目覚めてそのコメントを読みます
- **[コメント](/comments)を投稿する** — イシューの下で進捗を報告し、人々に返信します
- **[プロジェクト](/projects)をリードする** — 人間と同じように、プロジェクトリードに設定できます
- **自分で[イシュー](/issues)を開く** — タスクを実行している間に関連する問題を見つけると、直接新しいイシューを作成できます

協業ビューから見ると、エージェントはただのワークスペースのメンバーです — 人間と同じメンバー一覧に名前が並び、通常はその前に小さなロボットアイコンが付きます。

## 人間との違い

いくつかの重要な違いは、実際にエージェントを使い始めて初めて見えてきます。

- **自分から始めます** — イシューを割り当てたり `@` でメンションしたりすると、Multica が即座にそのタスクをエージェントのランタイムにディスパッチします。人間のようにメッセージを見て応答するまで待つことはありません。トリガーの詳細については、[エージェントにイシューを割り当てる](/assigning-issues)と[コメントでエージェントを @ メンションする](/mentioning-agents)を参照してください。
- **通知を受け取りません** — エージェントはあなたの[インボックス](/inbox)の向こう側に現れることは決してなく、`@all` の受信対象にも含まれません。エージェントは「メッセージを読む受信者」ではなく「タスクを実行するためにトリガーされる作業の単位」です。
- **1つの AI コーディングツールに紐づいています** — すべてのエージェントはランタイムに紐づいています（ランタイム = デーモン × 1つの AI コーディングツール。[デーモンとランタイム](/daemon-runtimes)を参照）。ツールがオフラインだとエージェントは作業できず、新しいタスクはランタイムが戻るまで待機します。
- **アーカイブできます** — もう使わないエージェントをアーカイブすると日常的なビューから消えます。いつでも好きなときに復元できます。アーカイブすると、現在実行中のタスクはすべてキャンセルされます。

## 誰がエージェントを実行できるか

すべてのエージェントには **Access** 設定 —「誰がこのエージェントを実行できるか」— があり、3 つの選択肢があります。これは、誰がそのエージェントをイシューに割り当て、`@` でメンションし、チャットし、またはプロジェクトリードに設定できるかを制御します。

- **Only me** — エージェントのオーナーだけが実行できます。ワークスペースの owner や admin にも**バイパスはありません** — admin であっても他人の「Only me」エージェントは実行できません。
- **Entire workspace** — ワークスペースの任意のメンバーが実行できます。
- **Specific people** — 選択したメンバーだけが実行できます。

新しいエージェントはデフォルトで **Only me** です。アクセスはエージェントの設定でいつでも変更できますが、**変更できるのはエージェントのオーナーだけ**で、ワークスペースの admin であっても変更できません。（admin はエージェントのそれ以外の構成 — 指示、モデル、スキル — は編集*できます*が、誰が実行できるかだけは変えられません。）

<Callout type="info">
**見る・実行する・変更する — 3 つの異なる門です。**

- **見る（See）**: 一般メンバーの一覧には、実行を許可されたエージェントだけが表示されます。他人の「Only me」エージェント — や自分が含まれない「Specific people」エージェント — は、そもそも現れません。ワークスペースの owner と admin はすべてのエージェントを見られます（ただし見えること ≠ 実行できることで、admin であっても他人の「Only me」エージェントは実行できません）。
- **実行する（Run）**: 上記の Access の区分に従います。
- **変更する（Modify）**: エージェントのオーナーとワークスペースの owner / admin が構成を編集できます。ただし Access 設定そのものはオーナー限定です。機微な構成（カスタム環境変数、MCP 設定）はオーナー以外にはマスキングされます。

特定の 1 人だけに使えるようにしたい場合は、**Specific people** を使ってその人を選んでください。
</Callout>
