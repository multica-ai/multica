title: 成员与权限
description: 邀请成员，并用 owner、admin 和 member 三种角色管理工作区权限。
---

import { Callout } from "fumadocs-ui/components/callout";

加入[工作区](/workspaces)的人都是成员。每位成员都有一个角色：`owner`、`admin` 或 `member`。角色主要控制工作区设置和团队管理；创建 issue、发表评论等日常协作对所有成员开放。

## 三种角色

| 操作 | owner | admin | member |
|---|:---:|:---:|:---:|
| 查看工作区并参与日常协作 | ✓ | ✓ | ✓ |
| 修改工作区设置 | ✓ | ✓ | — |
| 邀请 `admin` 或 `member` | ✓ | ✓ | — |
| 调整或移除 `admin`、`member` | ✓ | ✓ | — |
| 授予、收回或移除 `owner` | ✓ | — | — |
| 删除工作区 | ✓ | — | — |

工作区必须始终保留至少一名 `owner`，因此最后一名 `owner` 不能被降级、移除或直接离开。

<Callout type="info">
角色不决定谁能运行某个智能体。智能体使用独立的 Access 设置：Only me、Entire workspace 或 Specific people。即使是工作区的 `owner` 或 `admin`，也不能绕过 Access 运行未授权的智能体。详见[智能体](/agents#看见运行和修改是三种权限)。
</Callout>

## 邀请成员

`owner` 和 `admin` 可以输入邮箱并选择 `admin` 或 `member`，然后发送邀请。邀请不能直接授予 `owner`；需要先让对方加入工作区，再由现有 `owner` 提升角色。

对方不需要提前注册 Multica，但必须使用收到邀请的邮箱登录并接受邀请。邀请有效期为 7 天，过期后需要重新发送。

如果邮件发送失败，邀请记录仍然存在。管理员可以撤销邀请，也可以通过其他可信渠道把邀请链接交给对方。

## 调整角色

`admin` 可以在 `admin` 和 `member` 之间调整角色，也可以移除这两类成员。只有 `owner` 可以把成员提升为 `owner`，或更改、移除现有 `owner`。

角色变化立即生效。降低权限前，应确认对方不再需要管理工作区设置、成员或集成。

## 移除成员

成员被移除后会立即失去工作区访问权限。该成员已经创建的 issue、评论和其他协作记录仍保留在工作区中。

<Callout type="warning">
移除成员还会停用该成员拥有的运行时、归档绑定在这些运行时上的智能体，并取消尚未结束的执行任务。先完成运行时和智能体交接，再移除成员。
</Callout>
