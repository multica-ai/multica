This page walks you end-to-end through Multica Cloud — **sign up → install the [CLI](/cli) → start the [daemon](/daemon-runtimes) → create an [agent](/agents) → assign your first [task](/tasks)**. Takes about 5 minutes.

One prerequisite: you already have at least one [AI coding tool](/providers) installed locally ([Antigravity](/providers#antigravity), [Claude Code](/providers#claude-code), [CodeBuddy](/providers#codebuddy), [Codex](/providers#codex), [Cursor](/providers#cursor), [Copilot](/providers#copilot), [Grok](/providers#grok), [Hermes](/providers#hermes), [Kimi](/providers#kimi), [Kiro CLI](/providers#kiro-cli), [OpenCode](/providers#opencode), [OpenClaw](/providers#openclaw), [Pi](/providers#pi), [Qoder](/providers#qoder), [Trae CLI](/providers#trae), or [DevEco Code](/providers#deveco-code)). The daemon auto-detects them on startup and refuses to start if none are present.

## 1. Create an account

Sign up at [multica.ai](https://multica.ai). You can log in with email (6-digit verification code) or Google.

On first sign-in Multica walks you through a short onboarding wizard: a few optional questions, then **creating your workspace**, then **connecting a runtime**. The runtime step offers three paths — use this computer via the desktop app (recommended), connect from the terminal with the CLI (the steps below), or skip for now and connect later. If you connect a runtime during onboarding, Multica also sets up a starter agent (**Multica Helper**) and offers a few first tasks to assign to it — so you may reach step 6 without doing steps 2–5 by hand.

The rest of this page is the CLI path, which also works any time after onboarding.

## 2. Install the Multica CLI

**macOS / Linux (Homebrew recommended)**:

```bash
brew install multica-ai/tap/multica
```

**macOS / Linux (no Homebrew)**:

```bash
curl -fsSL https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.sh | bash
```

**Windows (PowerShell)**:

```powershell
irm https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.ps1 | iex
```

Verify the install:

```bash
multica version
```

## 3. Log in + start the daemon

A single command handles login and starts the daemon:

```bash
multica setup
```

`multica setup` will:

1. Configure the CLI to connect to Multica Cloud
2. Open your browser for login (same email verification code / Google OAuth as the web)
3. Store the generated PAT in `~/.multica/config.json`
4. **Start the daemon automatically** — it holds a WebSocket connection to the server, gets new tasks pushed in real time (with a 30-second polling fallback), and sends heartbeats every 15 seconds

<Callout type="info">
**Using the desktop app?** The desktop app **starts the daemon automatically** on launch — no need to run `multica setup` by hand. See [Desktop app](/desktop-app).
</Callout>

Verify the daemon is running:

```bash
multica daemon status
```

`online` means it has registered with the server.

## 4. Verify the runtime is online

In the web UI, open **Runtimes** in the sidebar (under the **Configure** group). The daemon you just started should appear as one or more active runtimes — one per AI coding tool installed locally. The **Add a computer** button on that page shows the same connection commands for any additional machine.

If it shows as offline, don't panic — see [Troubleshooting → Daemon can't connect to the server](/troubleshooting#daemon-cant-connect-to-the-server).

## 5. Create an agent

If onboarding already created **Multica Helper** for you, you can skip this step.

Otherwise, open **Agents** in the sidebar and click **+ New**. This opens the agent creation studio with two ways to start:

- **Start blank** — fill in the form yourself. Only two fields are required: a **name** and a **runtime** (which AI coding tool runs it). Description, instructions, skills, model, and access are optional.
- **Build with AI** (recommended) — describe the agent you want; an Agent Builder asks focused questions and prepares a live draft you can edit before creating.

Once created, the agent shows up in your workspace and can be assigned work like a human member. For every field in depth, see [Create and configure an agent](/agents-create).

## 6. Assign your first task

Create an issue in the web UI, or from the CLI:

```bash
multica issue create --title "Add an ASCII architecture diagram to the README"
```

Assign the issue to the agent you just created — click its avatar in the web UI, or use the CLI:

```bash
multica issue assign MUL-1 --to my-agent-name
```

`--to` takes the **name** of an agent or member. A substring match works — if the agent is called `my-code-reviewer`, `reviewer` resolves to it. If your workspace has overlapping names, pass `--to-id <uuid>` instead (mutually exclusive with `--to`); look up the UUID via `multica agent list --output json` or `multica workspace member list --output json`.

**What happens next from the daemon**:

1. The server pushes the task to it in real time, usually within seconds (status goes from `queued` to `dispatched`)
2. It invokes the matching AI coding tool to start work (status becomes `running`)
3. The AI works locally — it may read your code directory, run commands, edit files
4. When done, it reports the result back to Multica (status becomes `completed` or `failed`, depending on whether auto-retry kicks in)

The web UI updates in **real time** (via WebSocket) — no refresh needed.
