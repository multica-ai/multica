Skill 是工作区中可以复用的一组工作方法。它以 `SKILL.md` 为主文件，也可以附带脚本、模板和参考资料。

把 skill 添加到[智能体](/agents)后，智能体会在执行相关任务时使用其中的说明。相同的方法不必分别复制到每个智能体的指令中，修改时也只需要维护一份。

## Skill 和智能体指令有什么区别

| | 智能体指令 | Skill |
|---|---|---|
| **适合保存** | 这名智能体长期不变的职责、边界和交付要求 | 某类工作可复用的步骤、资料和工具 |
| **使用范围** | 只属于一名智能体，每次执行都会提供 | 可以添加给多名智能体，并单独启用或停用 |
| **例子** | "只审查前端代码，不直接修改" | 一套无障碍检查清单和报告模板 |

简单判断：内容是在定义"这名智能体是谁"，写进指令；内容是在说明"这类工作怎么做"，整理成 skill。

## 创建或导入 skill

在工作区的 **Skills** 页面选择"新建 skill"，可以使用三种方式：

| 方式 | 适合什么情况 |
|---|---|
| **手动创建** | 从空白的 `SKILL.md` 开始编写。 |
| **从 URL 导入** | 从 GitHub、ClawHub 或 Skills.sh 导入已经发布的 skill。 |
| **从运行时复制** | 扫描连接电脑上的 skill，并把选中的内容复制到工作区。 |

从运行时复制的是当时的文件快照。之后本地文件发生变化，不会自动更新工作区中的版本；需要重新导入或在 Multica 中编辑。

使用 CLI 时，也可以从 `.skill` 或 `.zip` 文件导入：

```bash
multica skill import --file ./review-helper.skill
```

导入时如果工作区已经存在同名 skill，可以停止导入、覆盖原内容、使用新名称保存，或跳过。覆盖只适用于你有权编辑的原 skill。

## Skill 中可以放什么

每个 skill 都包含一个主文件 `SKILL.md`。在这里写清：

- 什么时候应该使用这个 skill；
- 开始前需要检查什么；
- 建议按照哪些步骤执行；
- 结果应以什么形式提交；
- 哪些情况需要停止并向成员确认。

还可以添加支持文件，例如：

```text
SKILL.md
references/api-conventions.md
templates/review-report.md
scripts/check.sh
```

`SKILL.md` 是保留路径，不能再创建同名支持文件。支持文件会和主文件一起提供给智能体。

## 添加到智能体

创建或导入 skill 后，还需要把它添加到具体的智能体。你可以在 skill 页面选择智能体，也可以打开智能体的 **Skills** 标签页进行管理。

- 一名智能体可以使用多个 skill；
- 同一个 skill 可以供多名智能体使用；
- 暂时不需要时，可以停用绑定，而不必删除 skill；
- 修改后的内容从后续执行开始生效，不会改变已经运行中的任务。

只有有权修改该智能体的人，才能为它添加、移除或停用 skill。

## 工作区 skill 和仓库中的 skill

工作区 skill 保存在 Multica 中，适合由团队共同维护，并添加给不同智能体。

部分 AI 编程工具也会直接读取代码仓库中的项目级 skill，例如 `.claude/skills/` 或 `.agents/skills/`。这些文件仍由仓库管理，Multica 不会自动把它们登记为工作区 skill。具体支持方式取决于使用的工具，见 [AI 编程工具对照](/providers)。

## 谁可以修改 skill

任何工作区成员都可以创建或导入 skill。创建者和工作区 `owner`、`admin` 可以修改或删除它；其他成员可以查看和使用，但不能覆盖内容。

删除是永久操作，同时会从所有智能体上移除该 skill。

<Callout type="warning">
从外部导入的 skill 可能包含脚本、命令或不安全的指令。Multica 不会替你审查、签名或隔离这些内容。导入前请检查 `SKILL.md` 和所有支持文件，只使用可信来源。
</Callout>
