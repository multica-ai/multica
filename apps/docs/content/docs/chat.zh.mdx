对话适合还没有形成明确 issue 的工作：梳理想法、查询工作区情况、讨论方案，或者先让智能体完成一个小任务。

它和 issue 的主要区别不是智能体"看不到工作区"，而是这段交流**没有绑定某一条 issue**。智能体不会自动收到某条 issue 的描述和评论，但仍可以在权限允许的范围内使用 Multica CLI 查询工作区、项目、issue 和 skill。

## 开始对话

1. 打开侧边栏中的**对话**。
2. 点击新建，选择一个你有权运行的智能体。
3. 输入消息，也可以附加图片或文件。
4. 发送后，Multica 会为这条消息创建一次执行，回复会回到当前对话。

运行时离线时，消息会等待它重新上线。智能体被归档，或者你失去调用权限后，不能继续发送新消息。

## 对话还是 issue

| 需要 | 更适合 |
|---|---|
| 快速提问、探索方案或处理私人草稿 | **对话** |
| 明确负责人、状态、优先级和交付结果 | **Issue** |
| 让团队成员查看背景并继续讨论 | **Issue** |
| 临时查看工作区信息，不需要建立工作记录 | **对话** |

在 Multica 中创建的直接对话只对创建者开放，其他工作区成员和管理员不能读取。需要团队共同查看的结论，应整理到 issue、项目说明或 skill 中。

## 多轮上下文

同一段对话会尽量继续 AI 编程工具中的原会话，因此后续消息不需要重复前面的内容。Multica 会保存会话标识，并把后续执行路由到能够访问该会话的运行时。

如果本地会话已经不存在、上下文无法继续，或者上一次错误不适合复用，运行时会从新会话开始。对话中的历史消息仍然保留。

## 查看和停止执行

智能体运行时，界面会显示当前阶段和工具活动。展开**查看详情**可以检查更完整的执行过程。

需要中断时，点击输入框旁的停止按钮。已经返回到对话中的内容会保留；尚未开始处理的输入会尽量恢复到编辑框中。

## 整理对话列表

你可以重命名、置顶和归档对话：

- **置顶**把常用对话固定在列表前面；
- **归档**把对话移到"已归档"，并暂时设为只读；
- **取消归档**后可以继续发送消息；
- **删除**会永久移除对话及其中的消息。

<Callout type="warning">
删除对话无法撤销，并会停止其中尚未结束的执行。如果只是暂时不需要，优先使用归档。
</Callout>
