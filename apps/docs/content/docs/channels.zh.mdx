聊天集成让团队不用打开 Multica，也能直接向智能体提问、在群聊中 @它，或从聊天窗口创建 issue。

目前支持飞书/Lark 和 Slack。两种集成使用相同的会话、身份和执行机制，但安装方式不同。

## 选择平台

| | 飞书 / Lark | Slack |
|---|---|---|
| 安装方式 | 在 Multica 中生成二维码，用飞书或 Lark 扫码授权 | 在 Slack 创建 app，再把两个 token 填入 Multica |
| 私聊智能体 | 支持 | 支持 |
| 群聊或频道 | @ Bot 后触发 | @ Bot 后触发 |
| 创建 issue | `/issue` | `/issue` |
| 连接方式 | 平台长连接 | Socket Mode |

每个 Bot 绑定一个 Multica 智能体。需要在同一个聊天平台中使用多个智能体时，要分别连接多个 Bot。

详细步骤：

- [飞书 / Lark Bot 接入](/lark-bot-integration)
- [Slack Bot 接入](/slack-bot-integration)

## 一条消息如何执行

1. Multica 根据 Bot 找到对应的工作区和智能体。
2. 在群聊或频道中，只有明确 @ Bot 的消息会继续处理；私聊不需要 @。
3. Multica 校验发送者的账号绑定和工作区成员身份。
4. 消息进入一段[智能体对话](/chat)，并创建一次执行任务。
5. 智能体的回复回到原来的私聊或讨论串。

没有 @ Bot 的频道消息不会触发智能体，也不会被加入它的对话上下文。

## 对话如何区分

- 飞书/Lark 按聊天区分会话；同一个聊天中的后续消息会继续原会话。
- Slack 私聊按频道区分；频道中的每个 thread 分别保存一段会话。

在频道中继续追问时仍需再次 @ Bot。智能体只收到发给自己的消息，不会自动读取整个频道的历史。

## 第一次使用需要绑定账号

成员第一次向 Bot 发消息时，会收到一个账号绑定链接。登录 Multica 后，平台账号会和当前工作区成员关联。

绑定完成后，Multica 才会运行智能体。每条消息都会重新检查成员身份和智能体 Access；离开工作区或失去智能体调用权限后，不能继续通过 Bot 运行它。

<Callout type="info">
账号绑定只用于确认发送者身份。聊天平台中的其他成员不会因此自动加入 Multica 工作区。
</Callout>

## 管理连接

工作区 owner 和 admin 可以连接或断开 Bot，普通成员可以查看已经连接的集成并使用自己有权调用的智能体。

断开后，Bot 不再接收新消息。已有的 Multica 对话和执行记录仍然保留。

## 自托管

自托管部署需要先为对应平台配置一个 32 字节加密密钥，Multica 才会开放连接入口：

```dotenv
MULTICA_LARK_SECRET_KEY=<base64 编码的 32 字节密钥>
MULTICA_SLACK_SECRET_KEY=<base64 编码的 32 字节密钥>
```

这些密钥用于加密保存 Bot 凭据。生成、保存和轮换方法见[环境变量](/environment-variables)。Multica Cloud 已完成这项配置。
