Purpose: Verify that task completed and task failed IM notifications use the compact renderer, prefer `notification_summary`, and link to the final agent comment anchor.

Preconditions: The Multica web app is reachable. The user is signed in. The recipient user has active bindings for OpenClaw WeChat and DingTalk when delivery is being verified. The recipient has enabled `task_completed` and `task_failed` preferences for the relevant IM channel. At least one agent/runtime can complete and fail tasks, or a backend test fixture can emit equivalent `task:completed` and `task:failed` events with an agent final comment.

User flow: Create or reuse an issue assigned to the recipient. Trigger an agent task completion where the completion payload contains `notification_summary` such as "验收 PASS，编译通过，通知链路正常", and ensure the agent also posts a longer final comment. Verify the delivered WeChat and DingTalk notification content. Then trigger or simulate a task failure with a concise failure summary and verify the delivered WeChat and DingTalk notification content.

Expected results: For task completion, both WeChat and DingTalk compact messages are one-line IM-friendly summaries in the form `[任务完成] <agent> [OPE-xxx](...): <summary>`. The `<summary>` comes from `notification_summary` before falling back to the final comment. The issue link is embedded in the issue identifier and includes `?comment=<final_agent_comment_id>`. For task failure, both channels use `[任务失败]` compact formatting, use the failure summary, and also include the comment anchor when a final agent comment exists. DingTalk task delivery depends on `dingtalk + task_completed` or `dingtalk + task_failed`, not on `dingtalk + mentioned`.

Notes for automation: Full external delivery requires configured DingTalk and OpenClaw WeChat endpoints. When external services are unavailable, verify the server-created notification delivery payload snapshots, dispatcher-rendered content, and daemon delivery logs instead. The acceptance-critical observable strings are `[任务完成]`, `[任务失败]`, `notification_summary` text, and an issue markdown link containing `?comment=`.
