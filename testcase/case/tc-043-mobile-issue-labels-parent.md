Purpose: Verify that the mobile app issue detail screen supports editing labels, start date, and parent issue fields.

Preconditions: The Multica mobile app is running (or H5/responsive mobile view). The user is signed in. An issue exists with available labels configured in the workspace. At least one other issue exists that can serve as a parent.

User flow: Open the issue detail screen on mobile. Locate the issue properties/attributes section. Find the "Labels" / "标签" field and tap to edit. Select one or more labels from the available list. Confirm the selection and verify labels appear on the issue. Find the "Start Date" / "开始日期" field and tap to edit. Select a date using the date picker modal. Confirm and verify the start date is displayed on the issue. Find the "Parent Issue" / "父 Issue" field and tap to edit. Search or browse for another issue to set as parent. Select the parent issue and confirm. Verify the parent issue reference is displayed. Tap the parent issue field again and clear/change the parent.

Expected results: Labels can be added and removed from the issue detail on mobile. The label selector shows all workspace-configured labels. Start date can be set and cleared using a date picker. The date picker modal is mobile-friendly (touch-scrollable). Parent issue can be set by searching/selecting another issue. Parent issue can be changed or cleared after being set. All changes are persisted to the backend (verify on page reload). The issue detail renders correctly after each property change.

Notes for automation: These features are on the mobile issue detail screen (apps/mobile). The date picker is a modal component. Label selection may use a multi-select sheet or popover. Parent issue selection involves an issue search/picker. Use visible text labels like "标签", "开始日期", "父 Issue" to locate edit triggers.
