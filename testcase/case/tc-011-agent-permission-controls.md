Purpose: Verify that agent and runtime configuration enforce strict ownership-based permission controls — only the agent/runtime owner can edit configuration. Workspace admins can NOT edit other users' agents or runtimes (OPE-954). Non-owner members see read-only views.

Preconditions: The Multica web app is reachable. Three user roles are available: User A who owns an agent and a runtime, User B who is a workspace admin but not the agent/runtime owner, and User C who is a regular workspace member. The agent has environment variables, instructions, and skills configured.

User flow: Sign in as User C (regular member). Navigate to the Agents page and open the agent owned by User A. Navigate through the agent's tabs: Overview, Instructions, Environment, Skills. Observe whether the fields are editable or read-only. Then sign in as User B (admin). Open the same agent owned by User A. Verify that the admin also sees read-only views and cannot edit User A's agent configuration or runtime settings. Navigate to the Runtimes/Computers section. Open User A's runtime. Verify that admin cannot modify the runtime settings. Finally sign in as User A and open the same agent to confirm full edit access.

Expected results: When User C (regular member) views User A's agent: all tabs are read-only. When User B (admin) views User A's agent: all tabs are also read-only — admin privilege does NOT grant edit access to others' agents (OPE-954 fix). When User B (admin) views User A's runtime: the runtime settings are read-only and no edit/save controls are available. When User A views their own agent: all tabs are fully editable with save/update buttons visible. When User A views their own runtime: full edit access is available.

Notes for automation: Check for the presence or absence of edit controls (buttons, editable textareas, input fields) to determine read-only vs editable state. The key verification for OPE-954 is that an admin user CANNOT edit another user's runtime — previously admins had edit access to all runtimes, which was a permission bug. The Environment tab specifically should show keys but mask values for non-owners.
