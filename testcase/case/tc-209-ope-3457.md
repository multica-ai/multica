# TC-209: Channel 消息转 Issue 双向关联（OPE-3457）

## 关联信息

- **OPE 编号**: OPE-3457
- **Gitee PR**: 待补（feature 分支 `feat/ope-3457-channel-convert-issue` 尚未合入）

- **特性摘要**: Channel 消息可手工或经 Agent 转换为 Issue，转换后建立双向关联 —— 消息下方展示关联 Issue 芯片并跳转，Issue 侧记录来源线程，Issue 状态变更回灌线程系统消息；公开频道支持 @、频道分组排序。

## 验证要点

1. 消息转 Issue（手工 / 由 Agent）后，消息下方出现关联 Issue 芯片，点击跳转到该 Issue 详情页。
2. 关联 Issue 芯片的链接使用 Issue 编号格式（`MUL-123`），与站内其他 Issue 链接一致，不再是 UUID（Hash）格式。
3. 在关联 Issue 芯片上右键，弹出浏览器原生右键菜单（可“复制链接”），而非消息区域的自定义右键菜单；消息其余区域右键仍为自定义菜单。
4. Issue 状态变更后，来源线程内回灌对应状态系统消息。
5. 旧版本后端未返回 `identifier` 时，链接回退到 Issue UUID、显示回退到 `#编号`，不报错（API 兼容）。
6. Issue 详情页「来自频道讨论」徽章显示来源频道名（`来自 <频道名>`），点击跳转带 `?message=<源消息ID>` 深链，定位并高亮触发转换的那条消息；频道名缺失时回退「频道讨论」文案、链接回退到频道本身。
7. `GetIssue` 仅在详情端点 enrich `source_channel_name`/`source_message_id`（best-effort，互不依赖）；列表/更新/WS 广播路径不带这两个字段（omitempty），前端按可选字段读取。

## 备注

- `identifier` 字段后端由 `getIssuePrefix` + `Number` 拼装；前端按 API 兼容规则保留 `identifier ?? id` / `identifier ?? #number` 回退。
- `source_message_id` 取来源线程的 `root_message_id`（即触发转换的消息）；`source_channel_name` 取 `source_channel_id` 对应频道名。两者各自 best-effort 查询，单项失败不影响另一项。
