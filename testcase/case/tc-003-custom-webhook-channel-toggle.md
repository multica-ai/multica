Purpose: Verify that the Notifications settings page exposes one `Custom Webhook` master switch that saves and restores the webhook delivery state.

Preconditions: The Multica web app is reachable in a browser. The build under test includes the updated OPE-222 custom webhook notifications requirement where `Channels` no longer exposes separate switches for `@ mentions`, `assign`, and `subscribed issue updates`. The testcase auth fixture `testcase/auth/auth.json` is available and contains a valid fixed-code login for `tester@multica.com`. At least one custom webhook endpoint already exists for the signed-in user, either from testcase `tc-002-custom-webhook-create-and-test.md` or from equivalent setup data.

User flow: Sign in with the fixed verification code account and open the workspace `Settings` page on the `Notifications` tab. In the `Channels` section, locate the `Delivery channels` card and find the single row for `Custom Webhook`. Confirm the page does not split this channel into separate rows for `@ mentions`, `Assigned to me`, or `Subscribed issue updates`. Turn the `Custom Webhook` switch on, wait for the save state to finish, then refresh the page or revisit the `Notifications` tab. After confirming the enabled state persists, turn the same switch off and revisit the page again.

Expected results: The `Channels` section shows one `Custom Webhook` switch rather than three per-event switches. Enabling or disabling the switch does not show an error toast. After reload, the switch remains in the last state selected by the user, proving the aggregated webhook preference was saved. Inbox and DingTalk channel rows remain visible on the same page, showing the new webhook toggle is additive to the existing notification settings.

Notes for automation: Identify the correct control by the visible `Custom Webhook` label and nearby description text, not by DOM position. Treat any UI that still renders separate `@ mentions`, `Assigned to me`, or `Subscribed issue updates` switches for the webhook channel as a regression against the updated requirement.
